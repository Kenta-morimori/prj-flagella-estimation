## 追加仕様（このPhaseで必ず入れる）

### 要件 1-1-2: 未コミットの場合は実行を認めない
- `git status --porcelain` が空でない（未コミット変更あり）場合，CLI実行をエラー終了とする．
- エラー文には最低限以下を含める：
  - 「未コミット変更があるため実行を中止」
  - `git status --porcelain` の先頭N行（例：20行）を表示
  - 対処（commit/stash してから再実行）

#### 実装方針
- `init_run()` の早い段階でチェックし，dirtyなら `RuntimeError` 等で停止する．
- `manifest.json` / `run.log` は dirtyチェックの前に出さない（「禁止」なので成果物を残さない）．

#### 受け入れ基準（DoDに追加）
- 未コミット変更がある状態で `scripts/01_simulate_swimming.py` を実行すると，即エラー終了する．
- clean な状態なら従来通り実行できる．

### 要件 1-1-3: pre-commit を導入する（`.githooks/pre-commit` を活用）
※ここで言う “pre-commit” は Git hookとしての pre-commit を確実に動かす，の意味で定義する（Pythonの`pre-commit`フレームワーク導入とは別物）．

#### 要件
- `.githooks/pre-commit` を実際に動作させる（開発者の環境で有効化できる手順を用意）．
- pre-commit で最低限以下を実行する（CIと同等）：
  - `ruff format --check`
  - `ruff check`
  - `pytest`（テストがある場合）
- hookが失敗したら commit をブロックする．

#### 実装方針
- `git config core.hooksPath .githooks` を設定する方式で統一する．
- `scripts/setup_git_hooks.sh`（または `make hooks`）など，1コマンドで設定できる導線を追加する．
- READMEまたは `docs/` に「初回セットアップ手順」と「解除方法」を短く書く．

#### 受け入れ基準
- `core.hooksPath` 設定後，`git commit` で `.githooks/pre-commit` が起動し，ruff/pytest が失敗すると commit が止まる．
- CI（GitHub Actions）のチェック項目と矛盾しない．
