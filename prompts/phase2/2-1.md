## モデル定義（外部論文参照不要：ここに明記した式と値で実装する）

### 単位
- 内部計算はSI推奨（m, s, N, Pa*s, J）。
- YAMLは [µm] 入力が多いので、ModelBuilderでSIに変換して保持する。

### 幾何・離散化
- 菌体：中心線上のビーズ列（直線）。`n_body` は ds から自動決定。
- べん毛：各べん毛をビーズ列で表現。`n_flag` は ds から自動決定。
- フック：**3ビーズ**で表現（「菌体の付け根ビーズ」+「べん毛先頭2ビーズ」）。
- 初期形状：べん毛点列は「ヘリックス点列」で初期化してよい（ただし多形変換は幾何の直接切替ではなく、後述の平衡角スイッチで実現する）。

### トポロジ（前計算して保持）
以下のインデックス集合を `SimModel` に持たせる：
- spring（隣接ビーズペア）
- bending（三つ組：連続3ビーズ）
- torsion（四つ組：連続4ビーズ、二面角）
- hook（三つ組：菌体付け根 + べん毛先頭2）

## ポテンシャルと力（式をここに固定）

以下、各ビーズ位置を r_i、力ベクトルを f_i とする。

### (1) spring（Fraenkel型：強い伸長制限つき）
隣接ビーズ i-j（距離 d=|r_i-r_j|）に対し：
- 平衡長 L（body/flagellaそれぞれ。デフォルトは ds をSI変換した値）
- 伸長限界 s（>0）
- ばね定数 H

エネルギー：
U_s(d) = 0.5 * H * (d - L)^2 / (1 - ((d - L)^2)/(s^2))
有効域：|d-L| < s
実装は発散回避のため、|d-L| >= (1-eps)s の場合は (1-eps)s にクランプして計算する。

力：
F = - dU_s/dd * (r_i - r_j)/d を i に足し、jには -F を足す。

### (2) bending（三つ組 i-j-k）
j を中心に
- u = r_i - r_j, v = r_k - r_j
- theta = angle(u, v)
- 平衡角 theta0（polymorph状態で切替）
- 係数 k_b

エネルギー（cos形式で安定に）：
U_b = 0.5 * k_b * (cos(theta) - cos(theta0))^2
力はU_bの勾配で求める（安定な実装にする）。

### (3) torsion（四つ組 i-j-k-l）
bond(j,k)周りの二面角 phi を計算し、
- 平衡角 phi0（polymorph状態で切替）
- 係数 k_t

エネルギー：
U_t = 0.5 * k_t * (phi - phi0)^2
力は標準的な二面角ポテンシャルの勾配を実装する。

### (4) spring-spring 反発（交差防止）
隣接でない2つのセグメント（線分）S_a, S_b の最短距離 D を計算し、
D < cutoff のときのみ
U_ss(D) = A_ss * exp(-D / a_ss)
力はDを増やす方向へ作用する（近似でもよいが、数値的に安定な実装にする）。

### (5) hook 特別ルール（曲げ角しきい値）
hook三つ組（ib, j0, k1）について：
- theta_hook = angle(r_ib - r_j0, r_k1 - r_j0)

ルール：
- theta_hook > 90° のとき：hook曲げポテンシャルは **0（課さない）**
- theta_hook <= 90° のとき：交差回避のため bending を課す
  U_hook = 0.5 * k_b_hook * (cos(theta_hook) - cos(90°))^2
  （cos(90°)=0）

## モータートルク（最重要：力分配で実装）
トルクは「hook周辺ビーズに与える力分配」で実現する。必ず満たす：
- 総力 = 0
- 指定軸まわりの総トルク = ±T0（符号はrun/tumbleで切替）

実装要件（決定的な方法でOK）：
- モータ軸 e：べん毛先頭 bond（bead0->bead1）の単位ベクトル
- hook関連3ビーズ（body_attach, flag0, flag1）の位置からレバーアームを作り、
  例えば「flag0とflag1に±F、body_attachに調整力」を入れるなどして
  `sum r×f` が `±T0 * e` になるようにする（総力0も同時に満たす）。
- tumbleでは `reverse_n_flagella` 本を選び、そのべん毛のトルク符号を反転する。

---

## 多形変換（polymorph）：平衡角のスイッチとして実装
各べん毛の状態 state ∈ {normal, semicoiled, curly1} を持つ。
スイッチとは：
- bendingの theta0 と torsionの phi0 を state に応じて切替えること

run時：
- 全べん毛 state=normal、トルク符号=+（または定義したrun方向）

tumble時（反転べん毛のみ）：
- tumble開始：state=semicoiled
- tumble中間：state=curly1
- tumble終了：state=normal に戻す（configで変更可能にしてもよい）

---

## 流体力学（RPY）と運動方程式（overdamped）
更新式：
r(t+dt) = r(t) + (H(r) · f(r)) dt + xi

- f(r)：全ビーズ力（motor + spring + bending + torsion + repulsion + hook）
- H(r)：RPY mobility tensor（自由空間、HIあり）
  - i==j：
    H_ii = (1/(6*pi*eta*a)) * I
  - i!=j：RPYの標準式
    - r = |r_ij|, rhat = r_ij/r
    - r >= 2a の式 + r < 2a の正則化（必ず実装）

---

## Brownian項（任意、デフォルトOFF）
brownian.enabled=false：
- xi = 0（決定論）

brownian.enabled=true：
- xi はガウスで
  E[xi]=0
  Cov[xi] = 2 * kB * Temp * H(r) * dt
実装：
- sqrt(H) を作り、xi = sqrt(2*kB*Temp*dt) * sqrt(H) * z（z~N(0,I)）
- method：
  - "cholesky"：PSＤを仮定（必要なら jitter を対角に足す）
  - "eigh"：固有分解（負の固有値は0へクリップ）

## 出力互換（重要）
trajectory.csv 互換のために、各出力フレームで以下を作る：
- 位置：菌体中心（body beads平均または中点）

- v, omega：有限差分で推定

    ds_um: float

- body:
    length_total_um: float
    diameter_um: float

    viscosity_Pa_s: float

    torque_Nm: float

    spring: { H: float, s_um: float }

      kb: float

      kt: float

      cutoff_um: float

    enabled: bool
    kb: float

    tumble_tau: float
    semicoiled_tau: float
    curly1_tau: float

    enabled: bool  # default false

    fps_out: float
    duration_s: float

- render/output/seed:
