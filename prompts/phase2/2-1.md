## ゴール
Phase2の3D遊泳シミュレーションの「物理中身（モデリング/力学）」を、
Watari & Larson 2010 のモデルに沿う形で実装する。
ただし I/O（trajectory.csv、3D/2D出力、ディレクトリ構造）は維持する。

## 絶対に守る制約
- 既存の入出力互換（scripts実行フロー、出力ファイル、レンダの入口）は維持する。
- 物理実装は破壊的変更OK（toy physics撤去OK）。
- パラメータは `conf/sim_swim.yaml` で一元管理（コード内定数を極力排除）。
- wall補正（壁面のRPY補正）は v1 では実装しない（自由空間のみ）。

## 実装分割（この構成で実装）

## モデル定義（論文参照不要：式と入力をここで固定）

### 単位
- 内部計算はSI推奨（m, s, N, Pa*s, J）。
- YAMLのµm系入力はModelBuilderでSIへ変換して保持する。

### 離散化（重要：菌体とべん毛で方式が異なる）
- 菌体（body）：**n角柱×層**で表現（Fig.1相当）。
  - `body.prism.n_prism`（断面の角数。デフォルト3）
  - `body.prism.n_layers`（軸方向の層数）
  - `body.prism.dz_over_b`（層間隔。論文相当は0.5）
  - 1層あたり `n_prism` 個のビーズを正n角形として配置し、これを軸方向に `n_layers` 層積む
  - 総ビーズ数 `n_body = n_prism * n_layers`
- べん毛（flagella）：**長さ指定（ds）**でビーズ列を作る
  - 入力：`flagella.discretization.ds_over_b`（推奨）または `ds_um`
  - 内部計算：`n_flag = floor(L_flag / ds_flag) + 1`
- フック：各べん毛について **3ビーズ**（body付け根1 + flag先頭2）で表現

### べん毛付け根（attachment）
- 付け根は「菌体の端面（最終層）の断面ビーズ」から選ぶ
- `flagella.placement_mode: uniform` の場合：
  - 終端層の断面ビーズ集合から、べん毛本数分を均等に選択（n_prism=3, n_flagella=3なら3点すべて）

### トポロジ（前計算）
`SimModel` に以下のインデックス集合を保持：
- spring（隣接ビーズペア）
- bending（三つ組：連続3ビーズ）
- torsion（四つ組：連続4ビーズ、二面角）
- hook（三つ組：body_attach + flag0 + flag1）
- spring-spring反発評価用：セグメント集合（線分リスト）

## ポテンシャルと力（式をここに固定）

各ビーズ位置を r_i、力ベクトルを f_i とする。

### (1) spring（Fraenkel型：強い伸長制限）
隣接ビーズ i-j（距離 d=|r_i-r_j|）：
- 平衡長 L（body/flagellaそれぞれ。デフォルトは「配置間隔」）
- 伸長限界 s（>0）
- ばね定数 H

エネルギー：
U_s(d) = 0.5 * H * (d - L)^2 / (1 - ((d - L)^2)/(s^2))
有効域：|d-L| < s
実装は発散回避のため、|d-L| >= (1-eps)s の場合は (1-eps)s にクランプして計算する。

力：
F = - dU_s/dd * (r_i - r_j)/d を i に足し、jには -F を足す。

### (2) bending（三つ組 i-j-k）
j を中心に
- u = r_i - r_j, v = r_k - r_j
- theta = angle(u, v)
- 平衡角 theta0（polymorph状態で切替）
- 係数 k_b

エネルギー（cos形式）：
U_b = 0.5 * k_b * (cos(theta) - cos(theta0))^2
力はU_bの勾配で求める（安定な実装にする）。

### (3) torsion（四つ組 i-j-k-l）
bond(j,k)周りの二面角 phi を計算：
- 平衡角 phi0（polymorph状態で切替）
- 係数 k_t

エネルギー：
U_t = 0.5 * k_t * (phi - phi0)^2
力は標準的な二面角ポテンシャルの勾配を実装する。

### (4) spring-spring 反発（交差防止）
隣接でない2つのセグメント（線分）S_a, S_b の最短距離 D を計算し、
D < cutoff のときのみ
U_ss(D) = A_ss * exp(-D / a_ss)
力はDを増やす方向へ作用（近似可。ただし数値安定最優先）。

### (5) hook 特別ルール（曲げ角しきい値）
hook三つ組（ib, j0, k1）：
- theta_hook = angle(r_ib - r_j0, r_k1 - r_j0)

ルール：
- theta_hook > 90°：hook曲げポテンシャルは 0（課さない）
- theta_hook <= 90°：交差回避のため bending を課す
  U_hook = 0.5 * k_b_hook * (cos(theta_hook) - cos(90°))^2

## モータートルク（最重要：力分配で実装）
トルクは「hook周辺ビーズに与える力分配」で実現し、必ず満たす：
- 総力 = 0
- 指定軸まわりの総トルク = ±T0（符号はrun/tumbleで切替）

実装要件（決定的な方法でOK）：
- モータ軸 e：べん毛先頭 bond（bead0->bead1）の単位ベクトル
- hook関連3ビーズ（body_attach, flag0, flag1）に力を配り、
  sum r×f が ±T0*e になるようにする（総力0も同時に満たす）。

---

## 多形変換（polymorph）：平衡角のスイッチ
各べん毛の状態 state ∈ {normal, semicoiled, curly1} を持つ。
スイッチとは：
- bendingの theta0 と torsionの phi0 を state に応じて切替えること。

run時：全べん毛 state=normal
tumble時（反転べん毛のみ）：
- tumble開始：state=semicoiled
- tumble中間：state=curly1
- tumble終了：state=normal に戻す

---

## 流体力学（RPY）と運動方程式（overdamped）
更新式：
r(t+dt) = r(t) + (H(r) · f(r)) dt + xi

- f(r)：全ビーズ力（motor + spring + bending + torsion + repulsion + hook）
- H(r)：RPY mobility tensor（自由空間、HIあり）
  - i==j：H_ii = (1/(6*pi*eta*a)) * I
  - i!=j：RPY標準式（r>=2a と r<2a の正則化を必ず実装）

---

## Brownian項（任意、デフォルトOFF）
brownian.enabled=false：xi = 0
brownian.enabled=true：
- xi はガウスで E[xi]=0, Cov[xi] = 2*kB*Temp*H*dt（FDT整合）
- sqrt(H) を作り、xi = sqrt(2*kB*Temp*dt) * sqrt(H) * z（z~N(0,I)）
- method：cholesky / eigh（負固有値は0へクリップ、必要ならjitter）

## conf/sim_swim.yaml（2-1時点の要求）
- 2-1では「必要キーの定義と意味」まで要求する（値の確定は2-2で行う）
- 重要キーにはコメントで「意味・単位・論文対応」を書くこと

最低限必要なキー群：
- scale: b_um, bead_radius_a_over_b
- body.prism: n_prism, n_layers, dz_over_b, radius_over_b, axis
- body: length_total_um（派生でも良いが出力の説明に必要）
- flagella: n_flagella, discretization.ds_over_b, bond_L_over_b
- motor: torque_Nm, reverse_n_flagella
- potentials: spring(H_over_T_over_b, s), bend(kb_over_T, theta0_deg), torsion(kt_over_T, phi0_deg), repulsion(A_ss_over_T, a_ss_over_b, cutoff_over_b)
- hook: enabled, threshold_deg, kb_over_T
- run_tumble: run_tau, tumble_tau, semicoiled_tau, curly1_tau
- time: duration_s, dt_over_tau
- brownian: enabled, temperature_K, method, jitter
