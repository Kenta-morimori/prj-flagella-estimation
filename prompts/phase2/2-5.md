# Phase2: 力学パラメータ整理と単位系修正（kBTスケール撤廃 / BrownianはMVP外）

## 目的
bead-RPY 力学コアを Watari & Larson 2010 の無次元化に整合する形へ修正する。
MVP ではブラウン項を導入しない。ブラウン項はコメントで残す。

完了条件:
1. エネルギースケールが `motor.torque_Nm` に統一される。
2. 時間スケール `tau_s = viscosity_Pa_s * b_m^3 / motor.torque_Nm` が実装される。
3. 不要な config パラメータが削除され、参照箇所が全て追従修正される。
4. `uv run python -m scripts.01_simulate_swimming` がエラーなく実行できる。

## 修正方針（最重要）
### 1. ポテンシャル係数スケーリング基準を `kB*T` から `motor.torque_Nm` へ変更する
現状: `src/sim_swim/dynamics/engine.py` が `kB*T` を用いて spring/bend/torsion/repulsion の係数をスケーリングしている。
修正後: 係数スケーリングに `kB*T` を使用しない。係数スケーリングは `motor.torque_Nm` を使用する。

#### スケール定義（変数は最小限）
- `motor.torque_Nm` : トルクスケール [N·m]（論文の T に対応）
- `scale.b_m` : 長さスケール b [m]
- `fluid.viscosity_Pa_s` : 粘度 η [Pa·s]
- `tau_s = fluid.viscosity_Pa_s * (scale.b_m ** 3) / motor.torque_Nm` : 時間スケール [s]

#### 係数の SI 変換（必須）
config は論文 Table 相当の「無次元係数 `*_star`」を保持し、`engine.py` 初期化時に SI へ変換する。
以降、力計算は SI 単位で処理する。

- `elastic.kb_J   = elastic.kb_star   * motor.torque_Nm`          [J]
- `elastic.kt_J   = elastic.kt_star   * motor.torque_Nm`          [J]
- `repulsion.A_ss_J = repulsion.A_ss_star * motor.torque_Nm`      [J]
- `spring.H_N     = spring.H_star    * (motor.torque_Nm / scale.b_m)` [N]
- `repulsion.a_ss_m = repulsion.a_ss_star * scale.b_m`            [m]
- `spring.L_m     = spring.L_star    * scale.b_m`                 [m]
- `bead.a_m       = bead.a_star      * scale.b_m`                 [m]
- `repulsion.cutoff_m = repulsion.cutoff_star * scale.b_m`        [m]

---

### 2. Brownian 項は MVP に含めない
- 力学更新にブラウン項を加えない。
- ブラウン項の実装ファイルは削除しない。
- `src/sim_swim/dynamics/engine.py` の `step` 内でブラウン変位を加える処理は無効化し、コメントで「MVP外」と明記する。
- config から温度・ブラウン関連キーを削除する。

---

### 3. config を必要最小限へ再設計する
`conf/sim_swim.yaml` は以下のみを保持する。列挙されていないキーは削除する。

#### 必須キー
- `scale.b_m`
- `fluid.viscosity_Pa_s`
- `motor.torque_Nm`
- `integrator.dt_star`
- `bead.a_star`
- `spring.L_star`
- `spring.H_star`
- `spring.s`
- `elastic.kb_star`
- `elastic.kt_star`
- `repulsion.A_ss_star`
- `repulsion.a_ss_star`
- `repulsion.cutoff_star`
- `hook.theta_threshold_deg`

#### 削除キー
- 温度、`kB*T`、ブラウン項 enable、ブラウン項 seed
- `T = eta*b^3` のような上書き用キー
- torque を無視するキー

## 実装タスク（ファイルごと）
### A. `src/sim_swim/dynamics/engine.py`
1. `kB*T` を係数スケーリングに使用している処理を削除する。
2. `tau_s = viscosity_Pa_s * b_m^3 / motor.torque_Nm` を計算する。
3. `dt_s = dt_star * tau_s` を用いて位置更新を行う。
4. 上記「係数の SI 変換」を初期化時に実装し、力計算へ SI 係数を渡す。
5. ブラウン項は無効化し、コメントで「MVP外」と明記する。

### B. `src/sim_swim/dynamics/forces.py`
1. 入力係数が SI 単位であることを docstring に明記する。
2. 出力が力 [N] であることを維持する。
3. ねじり力の有限差分と角度ラップ処理は維持する。

### C. `src/sim_swim/dynamics/hydro_rpy.py`
1. `positions_m` と `bead.a_m` と `fluid.viscosity_Pa_s` を入力に取り、mobility の単位が `m/(N*s)` であることをコメントで明記する。
2. 返り値は `H`（shape: 3N x 3N）とする。

### D. `scripts/01_simulate_swimming.py`
1. `motor.torque_Nm` を無視する処理とログを削除する。
2. `tau_s` と `dt_s` と `duration_s` をログに必ず出力する。
3. 削除した config キー参照を全て修正する。

### E. `conf/sim_swim.yaml`
1. 「必須キー」以外を削除する。
2. `*_star` は無次元値として記述する。
