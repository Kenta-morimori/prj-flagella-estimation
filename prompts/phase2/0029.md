## 目的
本タスクの目的は、**菌体の形状維持のために使っている step ごとの `body rigid projection` を、最終的に廃止できる方向へ進めること**である。

現状では、菌体は step ごとに `body rigid projection` により剛体へ戻されている。  
これは論文モデルの「ポテンシャルにより nearly rigid を保つ」という思想と一致しない。

本タスクでは、まず以下を達成する。

1. `body rigid projection` を **config で ON/OFF 可能**にする
2. **菌体のみ**を対象とした shape stability test を整備する
3. projection OFF 条件で、菌体が **ポテンシャルのみ**で短時間安定に形状維持できるようにする
4. その結果をもって、今後「べん毛 projection がショットノイズ的に効いている可能性」を説明できる土台を作る

## 背景
論文モデルでは、body や flagella の shape は **spring / bending / torsion / repulsion** により *nearly rigid* に保たれる。  
一方、現状実装では body に対して step ごとに `body rigid projection` を適用しており、これは論文モデルからの差分である。

本リポジトリにおける現在の大目的は  
**「3本べん毛が菌体後方で束化し、推進する run 状態を再現すること」**  
である。

ただし、現時点ではべん毛 projection や basal link の扱いが束化阻害要因である可能性が高い。  
その因果を説明するには、まず **菌体側が projection なしでも十分安定**であることを示す必要がある。

そのため本タスクでは、**菌体のみ**を切り出して安定化を行い、次PRで扱う「べん毛 projection の影響切り分け」の前提を作る。

---

## 基本方針
### 1. 論文寄りの方針を優先する
まずは、**論文モデルと同型のポテンシャルで shape を維持できるか**を優先して確認すること。

- body spring
- body bending
- 既存の body 構造拘束
- 必要なら構造的な spring 追加

を先に検討し、**最初から単に係数を強くするだけの方針は避ける**こと。

### 2. 係数の強化は最終手段
`stiffness_scales` のような係数増しは、必要性が diagnostics と test で明確になった場合のみ最小限にとどめること。  
最初の解決策として採用しないこと。

### 3. 構造的拘束の追加は許容
菌体の shape maintenance に有効で、かつ論文モデルの「potentials で nearly rigid」に反しにくい対策は採用候補とする。

特に以下は検討対象としてよい。
- 三角柱の各側面長方形に対する **対角 bead 間 spring**
- 断面 shape を保つための最小限の構造的 spring

ただし、**実装するかどうかは diagnostics / test を見て判断すること**。  
推測だけで自動採用しないこと。

### 4. 初期条件は自然長・自然角とする
これまでの実装と同様に、**初期条件を自然長・自然角として扱う**こと。  
projection OFF 時に安定性を正しく評価するため、外力 0 の初期状態で内部力が極小になる構成を前提とすること。

## 仕様
## 1. `body rigid projection` を config で ON/OFF 可能にする
現状 `engine.py` の `step()` で常に適用されている `body rigid projection` を、config で切り替え可能にすること。

### 要件
- `SimulationConfig` に body projection 用の config を追加する
- default は **既存互換** とする
- つまり、初回実装では **default は ON のまま**
- CLI override で OFF にできること
- `with_overrides()` で自然に扱えること

### 命名
以下のような命名を推奨する。

```yaml
projection:
  enable_body_rigid_projection: true
````

既存の `projection` セクションに追加してよい。
ただし、既存キーとの整合を保ち、意味が曖昧にならないこと。

### 実装上の要件

* `enable_body_rigid_projection=true` のときは現状互換
* `enable_body_rigid_projection=false` のときは `_project_body_rigid()` を呼ばない
* body projection 以外の既存挙動は不用意に壊さないこと

---

## 2. body-only stability test を追加する

今回の主目的は、**菌体のみ**で projection OFF 時に shape stability が保てるかを検証することである。
したがって、べん毛なし (`n_flagella=0`) の test を追加すること。

### 必須テスト

#### (1) 外力なし静止平衡

* `n_flagella=0`
* motor torque = 0
* brownian = false
* body rigid projection OFF

で multi-step 実行し、以下を確認すること。

* body spring 長誤差が十分小さい
* body bending 角誤差が十分小さい
* NaN / inf が出ない
* COM drift が極小

#### (2) 一様外力下の並進

* `n_flagella=0`
* body rigid projection OFF
* 適度な一様外力を body に加える

このとき、

* body COM が移動する
* body shape は大きく崩れない
* spring / bending 誤差が許容範囲内

を確認すること。

#### (3) 回転 couple 下の回転

* `n_flagella=0`
* body rigid projection OFF
* body に純トルクに相当する外力対を加える

このとき、

* body が回転する
* body shape は大きく崩れない
* spring / bending 誤差が許容範囲内

を確認すること。

### 既存テストの再利用

以下は可能な限り再利用すること。

* `tests/test_simulation.py` の body constraint 系
* body spring / body bending drift の計測補助関数
* `tests/test_model_builder.py` の body geometry 初期検証

ただし、既存テストは projection ON 前提のものが多いため、**projection OFF 条件の専用 test は新設または明示拡張すること**。

---

## 3. body diagnostics CSV を追加する

菌体形状が崩れ始める原因を追えるように、短時間実行で全 step 記録可能な diagnostics CSV を追加すること。

### 出力方針

* 短時間（例: `time.duration_s <= 0.01`）では全 step 出力
* body-only の test / 実験で確認しやすい場所へ出力
* simulation の主挙動を変更しないこと

### 推奨ファイル

* `body_constraint_diagnostics.csv`
* 必要なら `body_constraint_summary.csv`

### `body_constraint_diagnostics.csv` の必須カラム

最低限、以下を含めること。

1. `step`
2. `t_s`
3. `body_spring_max_stretch_ratio`
4. `body_spring_mean_stretch_ratio`
5. `body_bend_max_error_deg`
6. `body_bend_mean_error_deg`
7. `body_triangle_area_min`
8. `body_triangle_area_max`
9. `body_centerline_max_deviation_um`
10. `com_x_um`
11. `com_y_um`
12. `com_z_um`

### 各量の意味

* `body_spring_*`: body spring の自然長からのずれ
* `body_bend_*`: body bending triplet の基準角からのずれ
* `body_triangle_area_*`: 各 layer の断面三角形面積の最小/最大
* `body_centerline_max_deviation_um`: layer centroid 列の直線性からの最大ずれ
* `com_*`: body center-of-mass

### 目的

この diagnostics の目的は、

* body rigid projection OFF で body がどこから崩れるかを可視化すること
* 将来、べん毛 projection 側の問題と切り分けること
  である。

## 4. 外力テストの考え方

`n_flagella=0` の body-only 条件では、外力は任意に設計できる。
ただし、**無意味に大きい値を入れて数値解法の限界だけを見ることは避ける**こと。

### 要件

* 外力強度は少なくとも 3 段階で比較すること

  * low
  * medium
  * high
* 少なくとも 1 段階は、body shape を保ったまま移動できる条件を見つけること
* どの強度を使ったか、最終報告と diagnostics に明記すること

### 注意

* dt は今回の主論点ではない
* まずは projection OFF で body の構造がどこまで耐えるかを見る
* それでも body 側だけで崩れる場合に限り、後続で dt の見直しを議論対象とする
