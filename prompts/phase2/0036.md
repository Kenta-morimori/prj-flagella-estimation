## 背景
現状の 3D plot では、力学モデル上は存在する接続のうち、body の一部 spring や hook が十分に明示されていない。
そのため、3D 可視化結果を見たときに、実装されている topology と視覚表現が一致していない。

このタスクでは、**実装済みの接続関係と 3D plot の表記を一致させること**を目的とする。

特に以下を満たすこと。
- body の spring が存在する箇所は全て 3D plot に表示する
- hook を他の部分と区別できるように黄色で表示する
- 既存の flagella 本体の色分け表示は維持する

## 目的
- 実装と可視化の不一致を解消する
- body の全 spring を 3D 表示に反映する
- hook を明示的に表示し、flagellum 本体との区別を明確にする
- `swim3d.mp4`, `swim3d_final.png`, `frames_3d/*.png` の表示を統一する

## スコープ
### このタスクでやること
1. body の spring が存在する全ての箇所を 3D plot に表示する
2. hook を黄色で表示する
3. 既存の flagella 本体の色分け表示は維持する
4. render 側の topology 取り扱いを整理し、実装と表示の不一致が起きにくい構成にする
5. テストを追加または更新する

### このタスクでやらないこと
- 力学モデルそのものの変更
- spring 定数や hook 定義の物理変更
- 2D 投影画像の仕様変更
- 無関係な refactor
- 無関係な config 追加
- 可視化だけの都合で topology を改変すること

## 要件
### 1. body の全 spring を表示する
ここでいう「body の全 spring」は、少なくとも以下を含むこと。
- body ring edges
- body vertical edges
- layer 間の diagonal brace spring（両対角）

3D plot 上で、これらがすべて描画されること。

### 2. hook の定義と表示
hook は、render 上では以下の 2 線分として扱うこと。
- `attach bead -> first flagellar bead`
- `first flagellar bead -> second flagellar bead`

すなわち、`attach-first-second` の 3 点に対応する 2 本の線分を hook として表示すること。

hook は **黄色** で描画し、flagellum 本体や body 線分と視覚的に区別できること。

### 3. flagellum 本体の表示
既存の flagella ごとの色分け表示は維持すること。

ただし、hook に含まれる区間については hook 色を優先し、flagellum 本体色で視認性が損なわれないようにすること。
必要であれば draw order や linewidth を調整してよい。

### 4. 出力対象
修正内容は、少なくとも以下の 3D 出力に反映されること。
- `swim3d.mp4`
- `swim3d_final.png`
- `frames_3d/*.png`（`save_frames_3d` 有効時）

## 実装方針
### 方針A: render 用 topology を明示する
3D 描画側で ad-hoc に線分を決め打ちするのではなく、render 用に必要な topology 情報を明示的に扱うこと。

少なくとも以下のどちらかの方針を採ること。
- `FlagellaRig` に body diagonal edges / hook segments を追加する
- render 用 helper を追加し、body / hook の描画線分を一元生成する

どちらでも良いが、**どの線分が body spring で、どの線分が hook か**がコード上で明確であること。

### 方針B: body spring の描画
body spring は現状の body 線分表示との連続性を保つため、灰系統で描画すること。
ring / vertical / diagonal が同系色で視認できればよい。

### 方針C: hook の描画
hook は黄色で描画すること。
body や flagellum 本体と十分に区別できるようにすること。

### 方針D: 重複描画の扱い
hook 区間は flagellum polyline と重複する可能性があるため、hook が見えるように表示順・線幅を調整してよい。
ただし、表示を複雑にしすぎないこと。
