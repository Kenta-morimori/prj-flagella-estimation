## 用語整理
### surrogate torque
actual motor torque ではなく、body の回転応答のみを切り分けるために与える**代理トルク入力**を指す概念名である。

### `body_equiv_load.pure_couple`
surrogate torque の具体実装の1つ。  
合力 0・非零トルクとなる 2 力を body に与える。Phase1 で用いる。

### torque split / force couple 分配
actual motor torque を、attach / first / second 近傍の bead 力へどう割り振るかの設計を指す。  
現在の主戦場の1つである。

### local constraint
hook / basal / first torsion 近傍に対して効く局所的な shape maintenance を指す。  
attach-first, first-second, second-third, basal bend, first torsion 近傍を主に扱う。

### canonical config
phase ごとに比較の基準として固定する標準設定。  
今回の比較は canonical config 上で行うこと。

### `minimal_basal_stub`
body + hook only ではなく、**3つの flagellar beads を含む短い basal stub** を持つ最小構成を指す。  
構造としては `attach(body) -> first -> second -> third` の最小 stub である。

### numerical finite
以下を満たして指定時間まで走ること。
- `pos_all_finite=True`
- `any_nan=False`
- `any_inf=False`

### shape-preserving finite
numerical finite に加えて、局所長さ・角度・torsion・motor split の異常が hard gate 内に収まっている状態を指す。

## 目的
今回の目的は、**1本べん毛条件で actual motor torque を入れたときの短時間安定化を Phase B / C に分けて実装し、before / after を diagnostics で説明可能にすること**である。

ただし、ここでいう「安定化」は **finite 完走だけ** を意味しない。  
**数値有限性 + 形状有限性** を満たすことを最終的な目標とする。
