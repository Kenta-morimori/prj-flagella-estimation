## 目的

## 用語・定義（固定）
- body長軸 `body_axis`：bodyビーズ座標のPCA第1主成分（単位ベクトル）
- 後方方向 `rear_dir`：`rear_dir = -body_axis`
- 基部接線 `tangent0`：`tangent0 = normalize(flag1_pos - flag0_pos)`
- 整列条件：`angle(tangent0, rear_dir) <= 10°`（初期状態で必須）

## 実装方針（最小破壊）
### 方針A（推奨・最小差分）
既存の「ヘリックス点列生成」を維持し、最終的にローカル座標を `axis_u/axis_v/axis_w` で回転して世界座標へ写す際の `axis_u` を `rear_dir` に置換する。

#### 具体手順
1) **body_axis（PCA）を計算**
- 入力：`body_positions_um`（shape=(N_body,3)）
- PCA：共分散→固有分解→最大固有ベクトルを `body_axis` に採用（単位化）

2) **rear_dir を定義**
- `rear_dir = -body_axis`

3) **局所基底（axis_u, axis_v, axis_w）を作る**
- `axis_u = rear_dir`
- `ref = ez=(0,0,1)`、もし `abs(dot(axis_u, ref)) > 0.9` なら `ref = ey=(0,1,0)`
- `axis_v = normalize(cross(ref, axis_u))`
- `axis_w = cross(axis_u, axis_v)`（右手系）

4) **既存のヘリックス点列（local_points）を axis_u/axis_v/axis_w で世界座標へ**
- 既存の `local_points` が「x方向が進行（接線）」の前提なら、
  - `world_points = origin + local_x*axis_u + local_y*axis_v + local_z*axis_w`
  として写す。

5) **接線整列の確認（生成直後）**
- `tangent0 = normalize(world_points[1] - world_points[0])`
- `angle(tangent0, rear_dir)` を計算し、10°を超える場合は例外（ValueError）で落とす（テストで必ず検出できるように）。
