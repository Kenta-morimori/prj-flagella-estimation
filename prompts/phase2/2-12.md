# Phase2 / 2-12: run状態を全時間で維持（スイッチング無効化）＋ multi-step tests

## 参照ドキュメント（必須）
- `prompts/00_project_context.md`
- `prompts/01_repo_rules.md`

## 対象
- Repo: Kenta-morimori/prj-flagella-estimation
- Branch: feature/phase2-fix-torque-force
- 論点: run/tumble（多形切替）を“必ず起こさない”設定を追加し、テストで担保する。

## 目的（決定）
- 「run状態維持」を **flagella の polymorph state が変化しないこと**として定義する。
- 全stepで `flag_state_changed == False` を満たす。

## 追加パラメータ（決定）
- `motor.enable_switching: false`（デフォルト false）
  - false: run/tumble更新（多形切替）を行わない（run固定）
  - true: 既存のrun/tumble更新ロジックを有効（現状の挙動）

## run固定のデフォルト（決定）
- Phase2 の MVP では **デフォルトで run 固定**とする。
  - 具体的には `motor.enable_switching` のデフォルト値を **false** に設定し、設定ファイル側で明示しない場合でも run 固定になるようにする。
- スイッチング挙動（run↔tumble / 多形切替）を検証する場合に限り、設定ファイルまたはCLI overrideで **`motor.enable_switching=true` を明示**する。

## 実装（最小破壊）
1) Config
- SimulationConfig に `motor.enable_switching` を追加して読み込めるようにする。
- デフォルト値は **false** とする（未指定でもrun固定）。

2) Switching更新ガード
- run/tumble（多形切替）更新処理の入口で以下を満たす：
  - `if not cfg.motor.enable_switching: return`（更新をスキップ）
- これにより、polymorph state が更新されず run 固定になる。

3) step_summary.csv へ列追加（テストを確実にするため）
- `flag_state_changed` を追加する。
  - `StepSummaryRecorder` が `model.flag_states` を前step分保持し、
    変化があれば True として記録する。

## テスト（必須 / multi-step / testsで検証）
- `tests/test_run_state_fixed.py` を追加する。
- テストの実行（検証）は **pytestで tests/ を実行すること**によって行う。
  - 手動で実行結果（動画・ログ等）を確認して合否判定しない。

### テスト要件
- 条件:
  - steps >= 1000（multi-step必須）
  - `motor.enable_switching=false`
- 検証:
  - 実行により生成された `step_summary.csv` において、`flag_state_changed` が全行 False

## DoD（完了条件）
- 新パラメータ `motor.enable_switching` が存在し、未指定でもデフォルトでrun固定（false）になる。
- 1000 step の pytest が安定して通る。
- 手動確認（動画目視等）を不要にする。
