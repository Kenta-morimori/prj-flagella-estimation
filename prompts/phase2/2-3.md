# Phase2 / 2-3 Codex Prompt: Δt・b・η・T入力 + Δt/τ=1e-3厳密 + 無次元時間で積分（duration=0.1s）

## 目的
Phase2-bead-rpy-core の実装に対して、時間の扱いを論文の無次元化（τスケール）に厳密に合わせる。
入力は Δt（実時間秒）, b, η, T としつつ、シミュレーションは必ず Δt/τ 刻み（= 1e-3）で回す。
デバッグのため、実時間の duration を 0.1 s に固定し、確実に完走できる状態にする。

## 適用範囲
- ブランチ：feature/phase2-bead-rpy-core
- 既存仕様（prompts/phase2/2-1.md, 2-2.md）は維持しつつ、**本プロンプトの指示を優先**する。

---

## 重要：時間スケールの定義（曖昧さ排除）
- τ（秒）は必ず以下で定義する：
  - `tau_s = viscosity_Pa_s * (b_m ** 3) / abs(motor.torque_Nm)`
  - `b_m = scale.b_um * 1e-6`
- ループで使用する無次元時間：
  - `t_star = t_s / tau_s`
  - `dt_star = dt_s / tau_s`

## 必須条件（この条件を満たさない設定はエラーにする）
- **dt_star は必ず 1e-3**（論文 Table 1 の time increment に合わせる）
  - すなわち `dt_s / tau_s == 1e-3`
- したがって、ユーザ入力の `time.dt_s` は
  - `dt_s == 1e-3 * tau_s`
  を満たさなければならない。
- 実装は起動時に以下を計算し、ズレがあれば ValueError とする：
  - `dt_expected = 1e-3 * tau_s`
  - `abs(dt_s - dt_expected) <= tol`（tol例：1e-15 秒、または相対誤差 1e-12）
- ログ（INFO）に必ず出力：
  - `b_um, viscosity_Pa_s, torque_Nm`
  - `tau_s`
  - `dt_s`
  - `dt_star (= dt_s/tau_s)`（=1e-3であること）
  - `duration_s`
  - `duration_star (= duration_s/tau_s)`
  - `total_steps = ceil(duration_star / dt_star)`

## 追加要件（必須）

### 1) time.dt_over_tau（入力キー）は不要（派生量としてのみ扱う）
- `conf/sim_swim.yaml` から `time.dt_over_tau` は削除する（入力として持たない）
- 代わりに `time.dt_s`（秒）を入力として持つ
- ただし上記の通り、`dt_s/tau_s == 1e-3` を満たさない場合はエラー

### 2) シミュレーションは必ず無次元時間で回す（Δt/τ刻み）
- Simulator/Dynamics の時間ループは無次元で管理する：
  - `t_star` を状態として持ち、`t_star += dt_star` で進める
- 物理計算で使う dt は「秒」へ戻した値：
  - `dt_s = dt_star * tau_s`
- 出力時刻は実時間で出す：
  - `t_s = t_star * tau_s`
- run/tumble の `*_tau` は **そのまま無次元時間（τ単位）**として扱い、
  - イベント判定は `t_star` で行う（秒換算しない）

### 3) body.length_total_um → body.n_layers 自動算出（0.5b刻み制約）＋ログ
- body.prism.n_layers はyamlで固定しない（削除/無視）
- 定義：
  - `L_over_b = body.length_total_um / scale.b_um`
  - `dz_over_b = body.prism.dz_over_b`（論文相当デフォルト 0.5）
  - 制約：`L_over_b` は `dz_over_b` の整数倍であること
- 算出：
  - `n_intervals = int(round(L_over_b / dz_over_b))`
  - `n_layers = n_intervals + 1`
- 誤差条件を満たさない場合は ValueError（曖昧さ排除）
- INFOログに `b_um, L_um, L_over_b, dz_over_b, n_layers` を出力
- ModelBuilder は算出した n_layers を用いて body prism beads を生成する

### 4) 起動時に「論文整合チェック」を行い、ズレがあればログに出す
- 目的：論文条件からの逸脱を早期検知
- チェック（最低限）：
  - `flagella.n_flagella == 3`
  - `scale.bead_radius_a_over_b == 0.1`
  - `potentials.spring.H_over_T_over_b == 10`
  - `potentials.spring.s == 0.1`
  - `potentials.bend.kb_over_T == 20`
  - `potentials.torsion.kt_over_T == 10`
  - `potentials.bend.theta0_deg == {142, 90, 105}`
  - `potentials.torsion.phi0_deg == {-60, 65, 120}`
  - `run_tumble.run_tau == 1200`, `tumble_tau == 800`, `semicoiled_tau == 400`, `curly1_tau == 400`
- 合致：INFO、不一致：WARNING（必要なら strict モードで ValueError）

## 実装タスク

1. `SimulationConfig/params` を更新し、tau_s を計算してログ出力。
2. dtの整合チェック：

   * dt_star = dt_s/tau_s を計算し、1e-3 と一致しない場合は ValueError。
3. シミュレーション時間管理：

   * ループは t_star を用い、dt_star=1e-3 で進める。
   * Dynamicsへ渡す dt は dt_s（= dt_star*tau_s）。
   * run/tumbleイベントは t_star と *_tau で判定。
4. body.n_layers の自動算出（0.5b刻み制約）＋ログ。
5. 論文整合チェックを追加（INFO/WARNING）。
6. `conf/sim_swim.yaml` を更新（duration=0.1、dt_s追加、dt_over_tau削除、n_layers削除）。
7. 既存I/O互換（trajectory.csv、3D/2D出力）は壊さない。
