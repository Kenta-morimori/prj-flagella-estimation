## 目的
1. `DynamicsEngine` 内の **τ（時間スケール）以外のスケーリングを `cfg.motor_torque_Nm` に統一**し、`torque_scale_Nm` は **τの計算にのみ使用**する。
2. **body–flag 接続を spring+hook にする**（距離拘束 + 角度拘束）。
3. **べん毛（hook/flagella）を菌体の中央層（`body_layers[len(body_layers)//2]`）から生やす**。

---

## 背景（前提）
- `motor.torque_Nm=0` のとき、実際に加えるモータトルクは 0 である（motor由来力はゼロ）。
- `torque_scale_Nm` は τ の基準スケール（例外時 τ=1 を成立させるための参照値）であり、**力学パラメータの絶対値を変える目的には使わない**。
- 現状 `bond_len_mean_body_flag_um` が NaN となるのは、body–flag が `spring_pairs` に存在せず `hook_triplets` のみで定義されているためである。

---

## DoD（完了条件）
- `torque_scale_Nm` は **τの計算にのみ使用**され、`DynamicsEngine` の `spring_h/k_bend/k_torsion/k_hook/repulsion_A` 等のスケーリングは **`cfg.motor_torque_Nm`（例外時は規約値）に基づく**。
- body–flag 接続について、`spring_pairs` に **(attach_idx, flag_base_idx)** が追加され、同時に `hook_triplets` も保持される（spring+hook）。
- attach 層が `body_layers[len(body_layers)//2]` に変更され、末端層からは生えない。

- torque=0 / hook.enabled=true/false の両方で、少なくとも **body–flag bond が存在**することを確認できる（ログ or 簡易assert）。

## タスク一覧（各タスクは別commit）

### Commit 1: τスケール分離（torque_scale_Nmはτのみ、他はmotor_torqueでスケール）
#### 変更方針
- `src/sim_swim/sim/params.py`
  - 既存の `torque_scale_Nm` と `tau_s` の設計は維持してよい（例外 {0,-1} で τ=1 が成立すること）。
  - `motor_torque_Nm` の定義（0,-1,通常）を明確にする（既存方針を踏襲）。
- `src/sim_swim/dynamics/engine.py`
  - `spring_h, k_bend, k_torsion, k_hook, repulsion_A ...` の計算に使っているスケール因子を **`cfg.torque_scale_Nm` から `abs(cfg.motor_torque_Nm)`（または規約値）へ置換**する。
  - 例外モード（`cfg.motor_torque_Nm in {0,-1}`）で `abs(cfg.motor_torque_Nm)` が 0 になってスケールが消えないように、**「力学スケール用トルク」 `torque_for_forces_Nm` を定義**して使うこと。
    - 推奨仕様：
      - `motor.torque_Nm == -1`: `torque_for_forces_Nm = torque_eta_b3_Nm`（既存規約に合わせる）
      - `motor.torque_Nm == 0` : `torque_for_forces_Nm = torque_eta_b3_Nm`（モータはOFFだが、剛性スケールは維持）
      - それ以外            : `torque_for_forces_Nm = abs(motor.torque_Nm)`
    - これにより「τのみ調整」「実トルク0でも剛性ゼロ化しない」を同時に満たす。

#### DoDチェック
- `engine.py` 内で `cfg.torque_scale_Nm` を参照している箇所が、**τ以外に残っていない**こと（grepで確認）。

### Commit 2: spring+hook（body–flag接続）を追加
#### 変更方針
- `src/sim_swim/model/builder.py`
  - 現状：`hook_triplets.append((attach_idx, idx[0], idx[1]))` のみ
  - 追加：**`spring_pairs.append((attach_idx, idx[0]))`**（body–flagの距離拘束）
  - rest length の扱いは既存の spring 実装に合わせる（L0が一様なら追加不要、配列管理なら追加必須）。
  - 二重拘束（spring+hook）になるため、安定性のために必要なら `k_hook`/`spring_h` の相対スケールを調整できるようにする（まずは既存値でよい）。

#### DoDチェック
- `step_summary_full.csv` の `bond_len_mean_body_flag_um` が NaN にならない（body–flagのボンド集合が空でない）。

---

### Commit 3: attach層を中央層へ変更（菌体中心から生やす）
#### 変更方針
- `src/sim_swim/model/builder.py`
  - 変更前：`_flag_attach_indices(body_layers[-1], n_flagella)`
  - 変更後：`_flag_attach_indices(body_layers[len(body_layers)//2], n_flagella)`
  - これにより、べん毛は菌体の中央層（胴体中央付近）から生える。

#### DoDチェック
- 生成された attach ビーズの z（または長手方向座標）が末端層ではなく中央付近にあること（ログ/簡易assert/可視化のいずれか）。
