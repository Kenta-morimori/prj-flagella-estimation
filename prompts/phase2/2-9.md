## 実装方針

### 1) フック長の維持
まずは spring（距離拘束）で維持している前提だが、長ステップで伸びるなら以下を導入する。

- 推奨：距離投影（constraint projection / PBD）
  - 各ステップ後に attach–flag0 の距離が 0.25b になるように補正する
  - 反復回数は 1〜5（設定可能、デフォルト2）
  - これにより「長時間でフックが離れる」を原理的に防ぐ

### 2) べん毛ボンド長（0.58b）の維持
- 初期化：べん毛点列は「隣接3D距離=0.58b」を満たすように生成する（00_project_context.md の仕様に従う）。
- 距離拘束：隣接 (i,i+1) spring の自然長を 0.58b に設定（または生成時点で距離=自然長となるよう保証）。
- 長ステップで drift が出る場合：
  - springを硬くするだけでなく、必要なら「隣接ボンドも距離投影」を導入して良い（ただし計算コストに注意）。

## tests/（pytest）での自動検証（手動禁止）

### Test A: Hook length multi-step（motor OFF）
条件：
- `motor.torque_Nm=0`
- `brownian=false`
- N>=2000 step（例：duration=2.0s, dt=1e-3）
期待（全ステップ）：
- `hook_count == n_flagella`
- mean：`|hook_len_mean - 0.25*b| <= 0.05*(0.25*b)`（±5%）
- min/max：`hook_len_min >= 0.25*b*(1-0.15)`, `hook_len_max <= 0.25*b*(1+0.15)`（±15%）

### Test B: Hook length multi-step（motor ON）
条件：
- `motor.torque_Nm=1e-18`
- `brownian=false`
- N>=2000 step
期待：Test A と同じ

### Test C: Flag intra bond length multi-step（motor OFF）
条件：
- `motor.torque_Nm=0`
- `brownian=false`
- N>=2000 step
期待（全ステップ）：
- `flag_intra_count > 0`（n_flagella=0なら skip）
- mean：`|flag_intra_len_mean - 0.58*b| <= 0.02*(0.58*b)`（±2%）
- min/max：`flag_intra_len_min >= 0.58*b*(1-0.05)`, `flag_intra_len_max <= 0.58*b*(1+0.05)`（±5%）

### Test D: Flag intra bond length multi-step（motor ON）
条件：
- `motor.torque_Nm=1e-18`
- `brownian=false`
- N>=2000 step
期待：Test C と同じ
