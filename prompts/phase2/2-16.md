## 背景
現状実装では、`body bead - first flag bead` の基部リンクが、論文モデルに即した

- 菌体長軸に対して垂直
- attach body bead が属する断面三角形の中心から attach 頂点 bead へ向かう方向
- 固定長

という条件を、初期状態から時間発展後まで一貫して満たしていない。

特に現状の `engine.py` における基部近傍の補正は、局所断面の

- **中心 → attach 頂点**

を使っておらず、実質的に

- **body 軸に垂直な平面への投影**
- **flag 全体の template projection による形状拘束**

に依存している。

したがって現状は、論文モデルに対して、**基部リンク方向の実装が未整合**であり、さらに **flag 形状保持の方法も論文より強い人工拘束を含んでいる**。  
本タスクでは、このうちまず **basal link 条件の前提不整合** を修正する。

## 目的（決定）
`body bead - first flag bead` を以下で固定する。

- **常に固定長**
- **その step における attach body bead が属する断面三角形の中心から attach 頂点 bead へ向かうベクトル方向**
- これは
  - 菌体長軸に対して垂直
  - 正三角形断面の中心→頂点方向の延長線上
  を意味する

また初期条件として、

- **`first flag bead - second flag bead` 以降の helix 軸は菌体後方へ向く**

ように配置する。

hook の自由度は引き続き

- `body bead - first flag bead - second flag bead`

の 3 点系に残す。

追加拘束を直接かけるのは **`first flag bead` まで** とし、

- `second bead` 以降は basal link 補正の**直接対象にしない**

## 実装方針（決定）

### 1. attach ごとの局所半径方向の定義
builder 側で各 flagellum の attach body bead について、その bead が属する body layer の幾何から以下を求める。

- attach body bead が属する body layer の重心
- `radial_unit = normalize(attach_vertex - layer_centroid)`

この `radial_unit` は

- `SimModel` に保持してよい
- もしくは各 step で body layer 幾何から再計算してよい

ただし、**step 後の補正で使う方向は、その時点の body layer 幾何に基づくもの**とする。  
すなわち、「body 軸に垂直であればよい」ではなく、**その step の断面中心→attach 頂点方向**を使うこと。

### 2. 初期配置の修正
初期配置生成時点で、各 flagellum について必ず

- `first_flag_bead = attach_body_bead + basal_rest_length * radial_unit`

を満たすこと。

ここで

- `basal_rest_length` は `body bead - first flag bead` の rest length
- 固定長は既存の spring rest length と整合してよい

さらに、

- `first bead - second bead` 以降の helix 軸は菌体後方へ向ける
- 第2 bead 以降は basal link 固定の直接対象にしない

### 3. post-step projection 順序の修正
時間発展中の post-step projection loop は、少なくとも概念上

1. `hook distance projection`
2. `basal_link direction fix`
3. `chain-length projection`

の順で適用する。

要求は以下。

- `body bead - first flag bead` を毎 step で **固定長・固定方向** へ戻す
- 固定方向は「body 軸に垂直な任意方向」ではなく、
  **その step の attach layer centroid → attach vertex**
- `second bead` 以降は basal link 補正の**直接対象にしない**
- hook 自由度は `body-first-second` の 3 点系として残す

### 4. 既存 template projection の扱い
現行の flag 全体テンプレート再投影が、`second bead` 以降まで basal link 補正の一部として直接動かしている場合、今回仕様と矛盾する。

そのため今回は、

- `body bead - first flag bead` の局所半径方向固定
- `flag chain` の長さ保持
- `flag 全体テンプレート再投影`

を**別の責務として分離して扱う**こと。

すなわち、**第2 bead 以降の配置そのものを basal link 補正の名目で直接テンプレートへ戻す実装は避ける**こと。

必要であれば既存 template projection は縮小・分離・無効化してよいが、今回の主目的はあくまで

- `first flag bead` の固定長・固定方向化

である。

### 必須確認項目

#### (1) 初期配置で `body bead - first flag bead` が固定長かつ断面中心→頂点方向である
各 flagellum について、初期状態で

- `||first - attach|| == basal_rest_length`
- `normalize(first - attach)` が
  `normalize(attach_vertex - layer_centroid)` と一致

を確認する。

#### (2) 初期配置で `first bead - second bead` 以降の helix 軸が菌体後方を向く
少なくとも初期接線、または `first-second` を使った局所軸が body rear direction を向くことを確認する。

#### (3) step 後も `body bead - first flag bead` の長さが保持される
1 step 後、および可能なら multi-step 後に

- `||first - attach|| == basal_rest_length`

が保持されることを確認する。

#### (4) step 後も `body bead - first flag bead` がその時の断面中心→頂点方向を向く
1 step 後、および可能なら multi-step 後に

- `normalize(first - attach)`
- `normalize(current_attach_vertex - current_layer_centroid)`

が一致することを確認する。

ここで使う方向は **その時の body layer 幾何** から求めること。

#### (5) `second bead` 以降は basal link 補正の直接対象ではない
少なくとも実装上・テスト上で、

- basal link 補正が直接書き換えるのは `first flag bead` のみ
- `second bead` 以降は chain-length / 他の既存拘束の影響は受けうるが、basal link 補正の直接ターゲットではない

ことを確認する。
