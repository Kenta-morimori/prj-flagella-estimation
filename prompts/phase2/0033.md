## 背景整理

### これまでに分かったこと
これまでの比較から、以下が分かっている。

#### 1. projection は本番解ではない
- `chain_length` / `template` を入れると短時間安定する
- しかしそれは **potential 主体の自然な安定化**ではなく、**強い人工拘束**の可能性が高い
- したがって、projection は本番モデルにも安定性評価にも使わない

#### 2. torsion finite difference 幅は主因の一部だった
- `paper_table1`
- `n_flagella=1`
- `motor.torque_Nm=0`
- all projection OFF

の条件で
- `fd_eps_over_b=0.1` では崩壊
- `fd_eps_over_b=1e-3`, `1e-4` では静止平衡をほぼ維持

だった。

したがって、
- 初期 mismatch や projection の有無そのものではなく
- **torsion force 実装の finite difference 幅**
が支配的だった可能性が高い。

現時点の current candidate は **`fd_eps_over_b=1e-3`** とする。

#### 3. motor split の gross bug は主因ではなさそう
- `Ta + Tb = Ttot` の residual は機械誤差レベル
- second bead 方向への過大分配はある程度緩和済み

なので、**motor split の単純な実装ミスが主因**とは言いにくい。

#### 4. body-only はかなり切れた
- baseline / `attach_proxy` はほぼ無変形
- old `pure_couple` は不適切な load distribution
- `pure_couple_distributed` では body-only short-time 条件で nearly rigid に近い応答

したがって、body 単独の一般的不安定性は主因候補から後退した。

#### 5. `minimal_basal_stub` と `full_flagella` で問題の層が違う
- body-only は短時間で安定
- `minimal_basal_stub` は短時間完走するが、`local_attach_first_rel_err` が大きい
- `full_flagella` では `local_first_torsion_err_deg` と `flag_bond_rel_err_max` が早期に立つ

したがって現時点では、

- **basal/hook 近傍の local stretch 問題**
- **full chain/torsion 側の追加問題**

の **二層構造**を仮説として保持する。
