# 2-14: template projection OFF 時に chain-length projection を併用できるようにする

## 目的
3 本べん毛・run 条件で `t=100` まで実行しても、論文 Fig.3 の Normal のような後方束化が再現されなかった。  
現状実装では、`src/sim_swim/dynamics/engine.py` において 1 step 更新後に flagellum 全体を初期テンプレート形状へ再投影する処理が入っており、これが束化に必要な相対配置の自由度を潰している可能性がある。

前タスクで template projection を config で OFF 可能にしたが、単純に OFF にすると flagella が飛散する挙動が確認された。  
このことから、現状の template projection は束化阻害だけでなく、flagellum の形状維持の一部も肩代わりしている可能性が高い。

そのため本タスクでは、**template projection を OFF にした場合のみ、flagellum の隣接 bead 間距離を保つ chain-length projection を併用できるようにし、比較実験可能な状態にする**。  
今回は **実装のみ** を対象とし、実験条件探索や他の物理変更は行わない。

## 背景と現状認識

### 1. 現状の post-step projection
`DynamicsEngine.step()` では、時間積分後に以下を実施している。

1. `self._project_body_rigid(pos_after)`
2. `self._project_hook_and_flag_bonds(pos_after)`

さらに `_project_hook_and_flag_bonds()` の内部で、各 iteration ごとに

- hook の距離拘束
- `_project_flagella_template()`

を適用している。

`_project_flagella_template()` は `self.flag_template_local` を用いて、各 flagellum を現在の基部位置と軸方向に合わせてテンプレート形状へ再配置する処理である。

### 2. 既に存在する chain-length projection
`engine.py` には `_project_flagella_chain_lengths()` が既に定義されている。  
これは各 flagellum の隣接 bead 間距離を rest length に合わせる処理である。

### 3. 今回の判断
- template projection は論文の力学更新式に明示的には含まれていない
- 一方、`_project_flagella_chain_lengths()` も厳密には論文そのものではなく、後処理 projection である
- しかし、template projection よりは拘束が弱く、「初期 helix 全体への再投影」を避けつつ最小限の chain 長拘束だけを与える

したがって今回は、**論文完全準拠の本採用ではなく、比較実験用の暫定措置として**  
**template projection OFF 時のみ chain-length projection を使う** 方針で実装する。

## 要件

### 1. config で制御可能にする
以下 2 つの状態が扱えるようにすること。

- 既存互換:
  - template projection ON
- 比較実験用:
  - template projection OFF
  - chain-length projection ON

要求:
- YAML/config から設定可能
- CLI override で切り替え可能
- `with_overrides()` で自然に扱えること
- デフォルトは **既存互換** を維持すること

命名は明確にすること。  
例:
- `projection.enable_flagella_template_projection: true`
- `projection.enable_flagella_chain_length_projection_when_template_off: true`

新規セクションを切る場合は、小さな追加に留めること。  
過剰な設計変更は不要。

---

### 2. engine 側の分岐を明示的に実装する
`src/sim_swim/dynamics/engine.py` を修正し、post-step projection を config に応じて分岐させること。

要求:
- template projection ON のとき:
  - **現状互換**
  - `_project_flagella_template()` を従来通り使う
- template projection OFF のとき:
  - `_project_flagella_template()` は呼ばない
  - 代わりに、必要に応じて `_project_flagella_chain_lengths()` を使う
- body rigid projection は維持すること
- hook の距離拘束は維持すること
- `constraint_projection_iters` の既存挙動は大きく壊さないこと

重要:
- **OFF 時に fly-away を防ぐため、chain-length projection を明示的に使うこと**
- **「template projection を切るだけ」で終わらせないこと**
