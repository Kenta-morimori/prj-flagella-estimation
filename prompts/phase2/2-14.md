# 2-13: template projection を config で OFF 可能にする

## 目的
3 本べん毛・run 条件で `t=100` まで実行しても、論文 Fig.3 の Normal のような後方束化が再現されなかった。  
現状実装では、`src/sim_swim/dynamics/engine.py` において 1 step 更新後に flagellum 全体を初期テンプレート形状へ再投影する処理が入っており、これが束化に必要な相対配置の自由度を潰している可能性がある。

そのため、本タスクでは **template projection を config で ON/OFF 可能にし、比較実験を行える状態にする**。  
今回は **実装のみ** を対象とし、実験条件探索や追加の物理変更は行わない。

---

## 変更対象
- `src/sim_swim/dynamics/engine.py`
- `src/sim_swim/sim/params.py`
- 必要なら:
  - `conf/sim_swim.yaml`
  - 関連テスト
  - 最小限のドキュメント更新

---

## 現状認識
`DynamicsEngine.step()` では、時間積分後に以下を実施している。

1. `self._project_body_rigid(pos_after)`
2. `self._project_hook_and_flag_bonds(pos_after)`

さらに `_project_hook_and_flag_bonds()` の内部で、各 iteration ごとに

- hook の距離拘束
- `_project_flagella_template()`

を適用している。  
`_project_flagella_template()` は `self.flag_template_local` を用いて、各 flagellum を現在の基部位置と軸方向に合わせてテンプレート形状へ再配置する処理である。:contentReference[oaicite:0]{index=0}

今回の比較実験では、**この template projection のみを OFF にしたい**。  
body の rigid projection や hook/bond 長関連の既存処理は、原則として維持すること。

---

## 要件

### 1. config で ON/OFF 可能にする
template projection の有効/無効を config で切り替えられるようにすること。

要件:
- デフォルトは **現状互換の ON**
- YAML から設定可能
- CLI override で切り替え可能
- `with_overrides()` で自然に扱えること

命名は明確にすること。  
例:
- `projection.enable_flagella_template_projection: true`

新規セクションを切る場合は、過度な設計変更は避け、小さな追加に留めること。

---

### 2. engine 側で OFF を反映
`src/sim_swim/dynamics/engine.py` を修正し、template projection の呼び出しを config で分岐すること。

要件:
- OFF 時は `_project_flagella_template()` を呼ばない
- ただし hook/bond 長に関する既存拘束は極力維持すること
- `constraint_projection_iters` の挙動は大きく壊さないこと

つまり、**OFF 時は flagella template への再投影だけを外す**。

---

### 3. 既存互換性
- デフォルト設定では現状と同じ挙動になること
- 既存 config を壊さないこと
- 既存の CLI 実行がそのまま通ること

---

### 4. テスト・確認
以下を行うこと。

- 既存 tests の実行
- 追加した設定が `False` のとき、template projection が skip されることの確認
- CLI override の具体例を最終報告に明記

必要なら最小限の test を追加してよい。  
ただし大規模な test 拡張は不要。

---

## 実装方針
- 最小変更で対応すること
- 不要な refactor はしないこと
- 今回は template projection の ON/OFF 化のみを扱うこと
- stiffness, repulsion, hook など他の物理パラメータには触れないこと

---

## 禁止事項
- template projection 以外の物理仕様変更
- stiffness scale の変更
- repulsion の変更
- 時間刻みや torque 系の変更
- ついでのリファクタ

---

## 期待する CLI override
実装後、以下のような切り替えができること。

```bash
uv run python -m scripts.01_simulate_swimming projection.enable_flagella_template_projection=false
````

セクション名を別にした場合は、最終報告で実際の override 形式を明記すること。
