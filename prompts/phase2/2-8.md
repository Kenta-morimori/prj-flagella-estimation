## 目的
1. **どの `motor.torque_Nm` でも数値安定性を担保**するため、常に `dt_star = Δt/τ = 1e-3` を満たすように **実時間Δt（内部計算ステップ幅）を自動調整**する。
   - 要件：**τは常に1** として扱い、`Δt = 1e-3` を成立させる（= dt_star=1e-3）。
   - 具体トルク指定時も、エラーではなく内部計算Δtを変えて安定化する。
2. **フック（body–flag接続）が正しく定義・分類されていることを検証**する（0.25bの距離、ボンド本数など）。
3. **菌体（body）を剛体に近づける**：距離(spring)+角度(bending)拘束を強化し、必要なら側面ブレース（斜めボンド）を追加する。
4. **テストを甘くしない**：multi-step（多数ステップ）で距離・角度制約が維持されることを必須にする。

---

## 背景と根拠
- overdamped + 明示更新 `x += (mobility@F)*Δt` のため、`dt_star` が大きいと発散する。
- これまで `motor.torque_Nm=1e-19` で発散した主因は、`dt_star` が 0.1 など大きくなっていたこと。
- 安定性の観点では「エラーで止める」より「内部Δtを調整して `dt_star=1e-3` を守る」ほうが運用しやすい。

## タスク（各タスクは別commit）

### Commit 1: dt_star固定（dt_star=1e-3）を常に成立させる
✅ 追加実装（仕様変更：最重要）

#### 要件
- τは常に 1 として扱う（無次元化のスケールを固定する運用）。
- 内部計算の実時間ステップ幅は常に `Δt = 1e-3 s` にする。
- つまり `dt_star = Δt/τ = 1e-3` を満たす。

#### 実装方針（最小破壊）
- 既存の `time.dt_s`（ユーザ指定）を「出力/保存間隔」として扱い、
  **内部の physics step は常に dt_internal_s = 1e-3** で回す。
- `duration_s` に対して `total_steps = round(duration_s / dt_internal_s)` を使う。

> 注意：もし既存コードが `tau_s` や `dt_star` を前提にした派生量を使っている場合は、まず「τ=1固定」に統一する（例：params側の dt_star検査を撤去/無効化）。

#### 追加ログ
- `tau_s`, `dt_internal_s`, `dt_star` を run.log と step_summary_full.csv の両方に出す（追跡可能にする）。

### Commit 2: hookの定義/分類を検証するログ拡張
✅ 追加実装（軽量）

#### 追加カラム（step_summary_full.csv）
- `[HOOK_BOND_STATS]`
  - `bond_count_body_flag`（body-flag springの本数）
  - `bond_len_mean_body_flag_um`
  - `bond_len_min_body_flag_um`
  - `bond_len_max_body_flag_um`

期待値（MVP）：
- `bond_count_body_flag == n_flagella`
- `bond_len_*_body_flag_um ≈ 0.25 * b_um`（b_um=1なら0.25µm）

#### もし期待値が外れる場合の切り分け
- 分類のバグか、そもそも spring_pairs に attach–flag0 が入っていないかを確定するため、
  body_flag に分類されたペアの `i,j` を（最初の数ステップだけ）別CSVに吐くのも可。

### Commit 5: テスト強化（multi-step必須、1e-19含む）
✅ 追加実装（pytest）

#### Test A: hook検証（静的）
- build直後に
  - `bond_count_body_flag == n_flagella`
  - `bond_len_mean_body_flag_um ≈ 0.25*b_um`（許容誤差例：±5%）

#### Test B: body rigidity multi-step（外力なし）
条件：
- `motor.torque_Nm=0`
- `brownian=false`
- `hook.enabled=false`
- `dt_internal_s=1e-3`（Commit 1 の固定運用）
評価（N=1000 steps 以上）：
- `max(|L-L0|/L0) < 1e-4`
- `max(|θ-θ0|) < 1e-3 rad`

#### Test C: motor on (1e-19) stability（必須）
条件：
- `motor.torque_Nm=1e-19`
- `brownian=false`
- `hook.enabled=true`
- `dt_internal_s=1e-3`（固定）
評価（例：N=2000 steps）：
- `pos_all_finite==True` を維持
- body の距離・角度誤差が Test B の閾値を満たす（もしくは少し緩めた閾値を設定）
