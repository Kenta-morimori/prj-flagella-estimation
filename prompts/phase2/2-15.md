## 背景整理

### 1. template projection
現状の `src/sim_swim/dynamics/engine.py` では、step 後の拘束処理の中で `_project_flagella_template()` を呼んでいる。  
これは各 flagellum の初期 local helix 形状を保存しておき、現在の基部位置と軸方向に合わせて **べん毛全体をテンプレートへ再配置する**処理である。

この処理は数値安定性には寄与するが、

- helix の横方向形状
- べん毛同士の局所配置
- hook 近傍の向き自由度

まで強く固定しやすく、論文の「hydrodynamics と hook により向きが決まる」構造を弱める可能性がある。

### 2. 論文寄りに形状を支える本筋
論文側では、形状維持は主に以下で支える。

- spring（隣接 bead 間距離）
- bending（局所曲率）
- torsion（局所ねじれ）
- hook
- spring-spring repulsion（非貫通性）

つまり、**形状はテンプレート再投影ではなく、局所ポテンシャルの積み上げで保つ**のが本筋である。  
論文は **半径拘束** や **位相進行拘束** を明示的には導入していない。  
それでも成り立つのは、FENE-Fraenkel spring・bending・torsion により各 flagellum が nearly rigid に保たれ、hook の柔らかさと hydrodynamics により全体 orientation が決まるからである。

### 3. 現状の問題
- template projection OFF だけでは fly-away する
- chain-length projection 併用でも即時凝集が起こる
- repulsion 強化でも即時凝集が残る
- stiffness scale を弱めても即時凝集が残る
- collapse onset diagnostics では距離急減が最初に起きている
- したがって、OFF 系はまだ比較実験可能な状態とは言えない

---

## 現時点の観測事実
以下は既に観測済みの事実として扱うこと。

1. `projection.enable_flagella_template_projection=true` では少なくとも短時間での破綻は抑えられる
2. `projection.enable_flagella_template_projection=false` かつ `projection.enable_flagella_chain_length_projection_when_template_off=false` では fly-away が起こる
3. `projection.enable_flagella_template_projection=false` かつ `projection.enable_flagella_chain_length_projection_when_template_off=true` では即時凝集が起こる
4. `potentials.spring_spring_repulsion.A_ss_over_T=5.0` と `potentials.spring_spring_repulsion.cutoff_over_b=0.3` にしても凝集が残る
5. `stiffness_scales.flag_bend_stiffness_scale=10`, `stiffness_scales.flag_torsion_stiffness_scale=10` にしても凝集が残る
6. collapse ケースでは `min_interflagella_distance_um` と `min_body_distance_um` が最初に急減する
7. stiffness scale の CLI override key は **`stiffness_scales.*`** を正とする
8. hook は **body bead – first flag bead – second flag bead** の 3 点系として扱う
9. body bead – first flag bead は **固定長リンク**として扱い、Phase5 では **菌体長軸に対して垂直方向に固定**する
10. 基部リンク方向固定は論文本文の明示要件ではなく、**比較実験用 / 数値安定化用の追加仮定**として扱う

この 10 点を前提として、以降のフェーズを設計すること。

## 必須出力ファイル 1: collapse_diagnostics.csv
以下のカラムを **必ず** 出力すること。

1. `step`
2. `t_s`
3. `flagellum_id`
4. `mean_radius_um`
5. `std_radius_um`
6. `mean_phase_diff_deg`
7. `std_phase_diff_deg`
8. `mean_bending_dev_deg`
9. `mean_torsion_dev_deg`
10. `hook_angle_deg`
11. `min_interflagella_distance_um`
12. `min_body_distance_um`

## 各カラムの定義
- `mean_radius_um`
  - 各 flagellum の基部から `max_flagella_points` bead を対象に、**attach body bead -> first flag bead** を軸として定義したときの、軸からの距離の平均
- `std_radius_um`
  - 上記距離の標準偏差
- `mean_phase_diff_deg`
  - 同じ対象 bead 群について、軸に垂直な平面で定義した位相角差 `phi_{i+1}-phi_i` の平均
- `std_phase_diff_deg`
  - 上記角差の標準偏差
- `mean_bending_dev_deg`
  - その flagellum に属する bending triplet について、平衡角 `theta0` からの偏差の平均
- `mean_torsion_dev_deg`
  - その flagellum に属する torsion quad について、平衡角 `phi0` からの偏差の平均
- `hook_angle_deg`
  - body 長軸と、`attach body bead -> first flag bead` ベクトルのなす角
- `min_interflagella_distance_um`
  - その flagellum の bead と、他 flagella の bead の最短距離
- `min_body_distance_um`
  - その flagellum の bead と、body bead の最短距離

## 必須出力ファイル 2: collapse_summary.csv
以下のカラムを **必ず** 出力すること。

1. `collapse_detected`
2. `first_collapse_step`
3. `first_collapse_t_s`
4. `flagellum_id_at_min_distance`

6. `global_min_body_distance_um`

- `first_collapse_step` と `first_collapse_t_s` は、その最初の step と時刻を記録する
- collapse が起こらなかった場合は
  - `collapse_detected=false`
  - `first_collapse_step=-1`
  - `first_collapse_t_s=-1.0`
  とする

## 実装仕様
- diagnostics の対象 bead 数は `max_flagella_points = 4`
- 対象 bead は各 flagellum の index `0,1,2,3`
- bead 数が 4 未満なら利用可能な全 bead

  - 時刻: `s`

# Phase 5: 基部リンク方向固定と bead-bead steric exclusion を導入する

## 目的
Phase4 までの結果から、template projection OFF 系では **`min_interflagella_distance_um` と `min_body_distance_um` の急減が最初に起きる**ことが分かった。  
また、repulsion や stiffness の調整だけでは collapse モードは変わらなかった。  
そのため Phase5 では、近接 collapse を直接抑えるために

1. **body bead – first flag bead の基部リンク方向を body 長軸に対して垂直に固定する**
2. **非結合 bead 間に bead-bead steric exclusion を追加する**

の 2 つを導入する。

### この方針の意味
- 基部リンク方向固定:
  - 第1べん毛 bead を body 局所法線方向に固定し、hook の自由度を **body bead – first flag bead – second flag bead** に集中させる
  - body bead – first flag bead は、body 剛体の一部そのものではなく、**body 剛体に従属する固定長・垂直方向リンク**として扱う
  - これは論文本文の明示仕様ではなく、**比較実験用 / 数値安定化用の追加仮定**
- bead-bead steric exclusion:
  - template projection のように全長形状を固定せず、**近づきすぎだけを抑える**
  - これは helix 形状維持そのものの保証ではなく、**近接 collapse 抑制のための補助**

## 5-1. basal_link セクションを追加する
`SimulationConfig` に `basal_link` セクションを追加する。  
default 値は以下に固定する。

```yaml
basal_link:
  enabled: false
  enforce_perpendicular_to_body_axis: true
  projection_alpha: 1.0
````

### 各パラメータの意味

* `enabled`

  * 基部リンク方向固定を有効化するか
* `enforce_perpendicular_to_body_axis`

  * body 長軸に対して垂直方向へ固定する
  * default は `true` で固定
* `projection_alpha`

  * 方向補正の適用率
  * Phase5 では **1.0 固定**を default とする

### 実装仕様

* `basal_link.enabled=true` のとき、**body bead – first flag bead** ベクトルを body 長軸に対して垂直へ補正する
* 補正は **固定長リンク**として行う
* 長さは `spring_rest_lengths_m` の hook 接続長を使う
* body 側 bead は動かさず、**first flag bead 側だけを補正する**
* 補正タイミングは **post-step projection loop 内** とし、順序は以下に固定する

```text
hook 距離拘束
→ basal_link 方向固定
→ chain-length projection
```

* `steric exclusion` は force 項として drift に含まれるため、post-step projection loop では適用しない
* basal_link 補正は **hook triplet 自体を削除しない**
* hook の曲がり自由度は **body bead – first flag bead – second flag bead** に残す
* 第2 bead 以降の shape は basal_link 補正の対象にしない

## 5-2. steric_exclusion セクションを追加する

`SimulationConfig` に `steric_exclusion` セクションを追加する。
default 値は以下に固定する。

```yaml
steric_exclusion:
  enabled: false
  epsilon_over_torque: 1.0
  sigma_over_2a: 1.0
  cutoff_over_sigma: 1.122462048309373
  exclude_same_flagellum: true
  exclude_body_body: true
  exclude_hook_neighbors: true
```

### 各パラメータの意味

* `enabled`

  * steric exclusion を有効化するか
* `epsilon_over_torque`

  * 反発強度。`epsilon = epsilon_over_torque * torque_for_forces_Nm`
* `sigma_over_2a`

  * WCA 型反発で用いる `sigma` を `2 * bead_radius_m` に対して何倍にするか
  * `sigma = sigma_over_2a * 2.0 * bead_radius_m`
* `cutoff_over_sigma`

  * cutoff 距離の倍率
  * default は `2^(1/6) = 1.122462048309373`
* `exclude_same_flagellum`

  * 同一 flagellum 内 bead 対は除外する
* `exclude_body_body`

  * body-body bead 対は除外する
* `exclude_hook_neighbors`

  * hook 近傍の結合隣接 bead 対を除外する

## 5-3. WCA 型 bead-bead steric exclusion を追加する

新しく **bead-bead の WCA (Weeks-Chandler-Andersen) 型反発**を追加する。
ポテンシャルは以下に固定する。

* `r < r_cut` のとき

  * `U(r) = 4*epsilon*((sigma/r)^12 - (sigma/r)^6) + epsilon`
* `r >= r_cut` のとき

  * `U(r) = 0`
* `r_cut = cutoff_over_sigma * sigma`
* 力はこのポテンシャルの負勾配を用いる

### 数値実装上の固定条件

* `r` の下限は `1e-12` m でクリップする
* 力の計算は bead-bead pair ごとに対称に加える
* 片方に `+f` を加えたら、もう片方に `-f` を加える
* 単位系は既存の SI ベースに合わせる
* `epsilon` と `sigma` は `DynamicsEngine.__init__` で前計算してよい

---

## 5-4. 反発を適用する bead 対を固定する

steric exclusion の対象 bead 対は、以下に固定する。

### 非結合 bead の定義

ここでいう **非結合 bead** とは、spring pair で直接つながっていない bead 同士を指す。
Phase5 で steric exclusion の対象とするのは、以下の対に固定する。

### 含める対

* 異なる flagella 間の bead-bead pair
* flagella と body の bead-bead pair

### 除外する対

* 同一 flagellum 内 bead-bead pair
* body-body bead-bead pair
* hook の結合近傍に対応する pair

  * attach body bead と first flag bead
  * first flag bead と second flag bead
  * attach body bead と second flag bead
* 完全に同一 index の pair

### 実装上の要件

* 除外対象 pair は `DynamicsEngine.__init__` で前計算する
* 反発対象 pair の index 配列を保持する
* 毎 step ごとに index 配列を組み直さない
* `steric_exclusion.enabled=false` の場合、力計算を完全にスキップする
