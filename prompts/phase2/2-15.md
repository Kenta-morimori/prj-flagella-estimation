# 2-15: template projection OFF 系を段階的に比較可能化する

## 目的
3 本べん毛・run 条件で `t=100` まで実行しても、論文 Fig.3 の Normal のような後方束化が再現されなかった。  
また、template projection を OFF にすると、

- chain-length projection なしでは発散
- chain-length projection ありでは即時凝集
- repulsion を `A_ss_over_T=5.0`, `cutoff_over_b=0.3` まで強めても凝集
- `flag_bend_stiffness_scale=10`, `flag_torsion_stiffness_scale=10` まで下げても凝集
- local-helix constraint を追加しても即時凝集

が確認された。

このことから、現状の template projection は束化阻害だけでなく、flagellum の形状維持や局所配置安定化の一部も肩代わりしている可能性が高い。  
一方で、template projection 自体は論文の力学更新式に明示的には含まれておらず、論文寄りの比較実験を行うには、OFF 系をより自然な拘束で支えられる状態へ段階的に移行する必要がある。

本タスクでは、**template projection OFF 系を比較可能な状態へ段階的に整備する**。  
変更は **フェーズ単位で明確に分けて実施**し、**ユーザーから次フェーズへの明示的な指示があるまで次に進まないこと**。

---

## 最重要ルール
- **フェーズをまたいで実装しないこと**
- **現在のフェーズで定義された範囲のみ実装すること**
- **各フェーズ完了後、停止して最終報告を行うこと**
- **ユーザーから「次のフェーズへ進んでよい」という明示的な指示があるまで、次フェーズの実装・追加修正・ついでの改善を行わないこと**
- **次フェーズのコード、test、config、ドキュメントを先回りして追加しないこと**
- **各フェーズで DoD を満たしたかどうかを明確に報告すること**

- **CLI override のキー名は、実装と一致させること**
- **曖昧な値を書かないこと**
- **「または」「必要なら」だけで仕様を逃がさないこと**
- **本プロンプト内の値・キー名・出力ファイル名を実装と一致させること**

## 背景整理

### 1. template projection
現状の `src/sim_swim/dynamics/engine.py` では、step 後の拘束処理の中で `_project_flagella_template()` を呼んでいる。  
これは各 flagellum の初期 local helix 形状を保存しておき、現在の基部位置と軸方向に合わせて **べん毛全体をテンプレートへ再配置する**処理である。

この処理は数値安定性には寄与するが、

- helix の横方向形状
- べん毛同士の局所配置
- hook 近傍の向き自由度

まで強く固定しやすく、論文の「hydrodynamics と hook により向きが決まる」構造を弱める可能性がある。

### 2. 論文寄りに形状を支える本筋
論文側では、形状維持は主に以下で支える。

- spring（隣接 bead 間距離）
- bending（局所曲率）
- torsion（局所ねじれ）
- hook
- spring-spring repulsion（非貫通性）

つまり、**形状はテンプレート再投影ではなく、局所ポテンシャルの積み上げで保つ**のが本筋である。  
論文は **半径拘束** や **位相進行拘束** を明示的には導入していない。  
それでも成り立つのは、FENE-Fraenkel spring・bending・torsion により各 flagellum が nearly rigid に保たれ、hook の柔らかさと hydrodynamics により全体 orientation が決まるからである。

### 3. 現状の問題
- template projection OFF だけでは fly-away する
- chain-length projection 併用でも即時凝集が起こる
- repulsion 強化でも即時凝集が残る
- stiffness scale を弱めても即時凝集が残る
- local-helix constraint を入れても即時凝集が残る
- したがって、OFF 系はまだ比較実験可能な状態とは言えない

---

## 現時点の観測事実
以下は既に観測済みの事実として扱うこと。

1. `projection.enable_flagella_template_projection=true` では少なくとも短時間での破綻は抑えられる
2. `projection.enable_flagella_template_projection=false` かつ `projection.enable_flagella_chain_length_projection_when_template_off=false` では fly-away が起こる
3. `projection.enable_flagella_template_projection=false` かつ `projection.enable_flagella_chain_length_projection_when_template_off=true` では即時凝集が起こる
4. `potentials.spring_spring_repulsion.A_ss_over_T=5.0` と `potentials.spring_spring_repulsion.cutoff_over_b=0.3` にしても凝集が残る
5. `stiffness_scales.flag_bend_stiffness_scale=10`, `stiffness_scales.flag_torsion_stiffness_scale=10` にしても凝集が残る
6. local-helix constraint を有効にしても即時凝集が残る
7. stiffness scale の CLI override key は `dynamics.*` ではなく **`stiffness_scales.*`** を正とする

この 7 点を前提として、以降のフェーズを設計すること。

# Phase 4: collapse onset diagnostics を導入し、OFF 系で何が壊れているかを定量化する

## 目的
template projection OFF 系で、何が先に壊れて即時凝集・即時発散に至るのかを定量化する。  
このフェーズでは、新しい拘束は追加しない。  
**diagnostics の追加のみ**を行い、collapse onset の原因を CSV と summary で観察できるようにする。

## 実施項目
- `diagnostics.collapse` セクションを新設する
- default は以下とする
  - `enabled = true`
  - `write_every_step = true`
  - `max_flagella_points = 4`
  - `collapse_distance_um = 0.15`
  - `collapse_consecutive_steps = 3`
- `outputs/.../collapse_diagnostics.csv` を出力する
- `outputs/.../collapse_summary.csv` を出力する
- `time.duration_s <= 0.01` の実行では **全 step** を必ず出力する
- `time.duration_s > 0.01` の実行でも `write_every_step=true` の場合は **全 step** を出力する
- diagnostics は simulation の既存挙動を変えない
- diagnostics は local-helix constraint を追加しない
- diagnostics は force や projection 経路に副作用を持たない

## 必須出力ファイル 1: collapse_diagnostics.csv
以下のカラムを **必ず** 出力すること。

1. `step`
2. `t_s`
3. `flagellum_id`
4. `mean_radius_um`
5. `std_radius_um`
6. `mean_phase_diff_deg`
7. `std_phase_diff_deg`
8. `mean_bending_dev_deg`
9. `mean_torsion_dev_deg`
10. `hook_angle_deg`
11. `min_interflagella_distance_um`
12. `min_body_distance_um`

## 各カラムの定義
- `mean_radius_um`
  - 各 flagellum の基部から `max_flagella_points` bead を対象に、**attach body bead -> first flag bead** を軸として定義したときの、軸からの距離の平均
- `std_radius_um`
  - 上記距離の標準偏差
- `mean_phase_diff_deg`
  - 同じ対象 bead 群について、軸に垂直な平面で定義した位相角差 `phi_{i+1}-phi_i` の平均
- `std_phase_diff_deg`
  - 上記角差の標準偏差
- `mean_bending_dev_deg`
  - その flagellum に属する bending triplet について、平衡角 `theta0` からの偏差の平均
- `mean_torsion_dev_deg`
  - その flagellum に属する torsion quad について、平衡角 `phi0` からの偏差の平均
- `hook_angle_deg`
  - body 軸と、`attach body bead -> first flag bead` ベクトルのなす角
- `min_interflagella_distance_um`
  - その flagellum の bead と、他 flagella の bead の最短距離
- `min_body_distance_um`
  - その flagellum の bead と、body bead の最短距離

## 必須出力ファイル 2: collapse_summary.csv
以下のカラムを **必ず** 出力すること。

1. `collapse_detected`
2. `first_collapse_step`
3. `first_collapse_t_s`
4. `flagellum_id_at_min_distance`
5. `global_min_interflagella_distance_um`
6. `global_min_body_distance_um`

## collapse 判定
- `min_interflagella_distance_um < 0.15` を **collapse 候補**とする
- それが **3 step 連続**した場合に `collapse_detected=true` とする
- `first_collapse_step` と `first_collapse_t_s` は、その最初の step と時刻を記録する
- collapse が起こらなかった場合は
  - `collapse_detected=false`
  - `first_collapse_step=-1`
  - `first_collapse_t_s=-1.0`
  とする

## 実装仕様
- diagnostics の対象 bead 数は `max_flagella_points = 4`
- 対象 bead は各 flagellum の index `0,1,2,3`
- bead 数が 4 未満なら利用可能な全 bead
- body 軸は、既存 builder / body geometry と整合する長軸を使用する
- 位相差は `wrap_angle` を使って `[-pi, pi)` に正規化する
- 単位は CSV 上では
  - 長さ: `um`
  - 角度: `deg`
  - 時刻: `s`
- diagnostics 計算は simulation step 後の positions を使う
