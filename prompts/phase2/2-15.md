# 2-15: template projection OFF 系を段階的に比較可能化する

## 目的
3 本べん毛・run 条件で `t=100` まで実行しても、論文 Fig.3 の Normal のような後方束化が再現されなかった。  
また、template projection を OFF にすると、

- chain-length projection なしでは発散
- chain-length projection ありでは即時凝集
- repulsion を `A_ss_over_T=5.0`, `cutoff_over_b=0.3` まで強めても凝集
- `flag_bend_stiffness_scale=10`, `flag_torsion_stiffness_scale=10` まで下げても凝集

が確認された。

このことから、現状の template projection は束化阻害だけでなく、flagellum の形状維持や局所配置安定化の一部も肩代わりしている可能性が高い。  
一方で、template projection 自体は論文の力学更新式に明示的には含まれておらず、論文寄りの比較実験を行うには、OFF 系をより自然な拘束で支えられる状態へ段階的に移行する必要がある。

本タスクでは、**template projection OFF 系を比較可能な状態へ段階的に整備する**。  
変更は **フェーズ単位で明確に分けて実施**し、**ユーザーから次フェーズへの明示的な指示があるまで次に進まないこと**。

## 背景整理

### 1. template projection
現状の `src/sim_swim/dynamics/engine.py` では、step 後の拘束処理の中で `_project_flagella_template()` を呼んでいる。  
これは各 flagellum の初期 local helix 形状を保存しておき、現在の基部位置と軸方向に合わせて **べん毛全体をテンプレートへ再配置する**処理である。

この処理は数値安定性には寄与するが、

- helix の横方向形状
- べん毛同士の局所配置
- hook 近傍の向き自由度

まで強く固定しやすく、論文の「hydrodynamics と hook により向きが決まる」構造を弱める可能性がある。

### 2. 論文寄りに形状を支える本筋
論文側では、形状維持は主に以下で支える。

- spring（隣接 bead 間距離）
- bending（局所曲率）
- torsion（局所ねじれ）
- hook
- spring-spring repulsion（非貫通性）

つまり、**形状はテンプレート再投影ではなく、局所ポテンシャルの積み上げで保つ**のが本筋である。  
論文は **半径拘束** や **位相進行拘束** を明示的には導入していない。  
それでも成り立つのは、FENE-Fraenkel spring・bending・torsion により各 flagellum が nearly rigid に保たれ、hook の柔らかさと hydrodynamics により全体 orientation が決まるからである。

### 3. 現状の問題
- template projection OFF だけでは fly-away する
- chain-length projection 併用でも即時凝集が起こる
- repulsion 強化でも即時凝集が残る
- stiffness scale を弱めても即時凝集が残る
- したがって、OFF 系はまだ比較実験可能な状態とは言えない

---

## 現時点の観測事実
以下は既に観測済みの事実として扱うこと。

1. `projection.enable_flagella_template_projection=true` では少なくとも短時間での破綻は抑えられる
2. `projection.enable_flagella_template_projection=false` かつ `projection.enable_flagella_chain_length_projection_when_template_off=false` では fly-away が起こる
3. `projection.enable_flagella_template_projection=false` かつ `projection.enable_flagella_chain_length_projection_when_template_off=true` では即時凝集が起こる
4. `potentials.spring_spring_repulsion.A_ss_over_T=5.0` と `potentials.spring_spring_repulsion.cutoff_over_b=0.3` にしても凝集が残る
5. `dynamics.flag_bend_stiffness_scale=10`, `dynamics.flag_torsion_stiffness_scale=10` にしても凝集が残る

この 5 点を前提として、以降のフェーズを設計すること。

# Phase 4: 基部近傍限定の local-helix constraint を導入する

## 目的
template projection OFF 系で、全長テンプレート拘束に頼らずに **helix の横方向形状を最小限維持**しつつ、全体の orientation と bundle 形成は hydrodynamics / hook に委ねる比較実験基盤を作る。

この local-helix constraint は **論文モデルそのものではない**。  
現実装では、論文モデルが想定する「局所ポテンシャルだけで nearly rigid を保つ regime」にまだ入っておらず、template projection OFF で即時発散または即時凝集が起こるため、**その regime に近づけるための最小限の数値補助**として導入する。

## 実施項目
- `local_helix` セクションを新設する
- default は以下とする
  - `enabled = false`
  - `n_local = 4`
  - `k_radius_over_torque = 0.5`
  - `k_phase_over_torque = 0.5`
- 各 flagellum の**基部から 4 bead**を対象に local-helix constraint を作用させる
- local-helix constraint は以下 2 成分で構成する
  1. **半径拘束**
     - 各対象 bead の軸からの距離 `rho_i` を初期 helix 半径 `r0` に近づける
  2. **位相進行拘束**
     - 対象 bead 間の軸回り角差 `phi_{i+1}-phi_i` を初期 helix の角差 `Δphi0` に近づける
- 強度は `torque_scale_Nm` を用いてスケールし、`engine` 内部で
  - `k_radius = k_radius_over_torque * torque_for_forces_Nm / max(b_m, 1e-30)`
  - `k_phase = k_phase_over_torque * torque_for_forces_Nm`
  とする
- `template projection OFF` のときのみ local-helix constraint を有効にできる設計にする
- `template projection ON` では local-helix constraint を適用しない
- `chain-length projection` は残す
- `template projection` と `chain-length projection` は削除しない

## 実装仕様
- local-helix constraint の対象 bead は **各 flagellum の index 0,1,2,3** の 4 bead とする  
  ただし flagellum bead 数が 4 未満の場合は **利用可能な全 bead** を対象とする
- 軸は **attach body bead -> first flag bead** の方向で定義する
- 半径は、その軸に垂直な平面での距離で定義する
- 位相角は、その平面上の基底で定義する
- `wrap_angle` を使って位相差を `[-pi, pi)` に正規化する
- force 実装は有限差分ではなく、**まずは position correction 型の弱い post-step projection として実装してよい**
- ただし correction は **対象 bead のみ** に作用させる
- correction は `constraint_projection_iters` のループ内で毎回適用し、`hook 距離拘束 -> chain-length projection -> local-helix correction` の順で適用する
- correction 1 回あたりの係数は
  - `alpha_radius = 0.2`
  - `alpha_phase = 0.2`
  とし、config には出さない
- `template projection OFF` かつ `local_helix.enabled=true` のときのみ local-helix correction を使う
