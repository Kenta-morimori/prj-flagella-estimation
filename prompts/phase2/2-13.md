## 目的（決定）
### 1) べん毛本数
- 物理モデルの `flagella.n_flagella` を **0〜9本** に拡張する。
- 付着候補は **中心層の三角形 + その前後に隣接する三角形** の合計 **9候補** とする。
- **各候補点には最大1本まで** とする。

### 2) 付着位置の選び方
- **1〜3本**: 従来互換を維持し、**中心三角形の3点のみ** を使う。
- **4〜9本**: 9候補から **重複なしランダム抽出** とする。
- ランダム性は **既存の乱数seed (`seed.global_seed`)** に従って再現可能であること。

### 3) 2D投影像
- 擬似顕微鏡像（2D投影）では、**菌体の重心が常に画像中心** に来るようにする。
- これは **レンダリング時の平行移動のみ** で実現し、物理座標そのものは変更しない。
- すなわち：
  - `trajectory.csv` や 3D状態はそのまま
  - 2D投影動画だけ中心固定
- パラメータで切替可能にし、**デフォルトはON** とする。

### 4) 3D動画の状態明記
- 3D動画の各フレーム左上に、状態ラベルとして **`RUN` または `TUMBLE`** を表示する。
- 2D動画には不要。
- `motor.enable_switching=false` の場合も `RUN` が明示表示されること。
- Polymorph の詳細状態（`normal/semicoiled/curly1` や flagella ID）は表示不要。

---

## 重要な解釈（固定）
### RUN / TUMBLE の定義
本タスクでは動画表示上、以下で十分とする。

- **RUN**:
  - 反転べん毛が存在しない、または
  - `motor.enable_switching=false` で実質 run 固定、または
  - 全flagellumが `PolymorphState.NORMAL`
- **TUMBLE**:
  - `motor.enable_switching=true` かつ、
  - 少なくとも1本の reverse flagellum が `SEMICOILED` または `CURLY1` に入っている状態

実装は、既存の `flag_states` / `reverse_flagella` / `enable_switching` を用いて最小追加で判定すること。

## 実装方針（最小変更）
### 1) べん毛付着候補の拡張
主に `src/sim_swim/model/builder.py` を修正する。

#### 現状
- 中心層 `center_layer` の3点のみを候補としている。

#### 変更
- `body_layers` から
  - `center_idx = len(body_layers) // 2`
  - `center_idx - 1`
  - `center_idx`
  - `center_idx + 1`
  の3層を候補として使う。
- それぞれ3点ずつ、合計9候補を得る。
- ただし **1〜3本は center_layer のみ** を使用する。
- **4〜9本は 9候補から非復元抽出**する。

#### 注意
- 層数が十分でないケースでも安全に動くようにする。
- MVP前提では通常3層以上ある想定だが、index境界は必ず保護する。
- `placement_mode` は既存互換を壊さない範囲で使う。今回必要以上に拡張しない。
- 既存の attach 順序に依存するテストがあれば、最小修正で通す。

### 3) 2D中心固定投影
主に `src/sim_swim/render/project2d.py` を修正する。

#### 方針
- 現在の `cam_center` は
  - `follow_camera_2d=true` なら `st.position_um[:2]`
  - false なら `(0, 0)`
  になっている。
- 今回はこれと独立に、**菌体重心を画像中心に置くモード**を追加する。

#### 実装
- `center_body_in_2d=true` の場合は、各 state の **菌体ビーズ重心** を `cam_center` に使う。
- ここでの重心は `SimulationState.position_um` を使って良い。
  - これは `_observe()` で body center から作られているため。
- `center_body_in_2d=false` の場合は従来ロジックを維持する。

#### 優先順位
- `center_body_in_2d=true` を最優先にして良い。
- つまり 2Dでは:
  1. `center_body_in_2d=true` → body center固定
  2. それ以外 → 従来の `follow_camera_2d` 判定
- 無理に複雑な仕様へしないこと。

---

### 4) 3D動画の RUN / TUMBLE 表示
主に `src/sim_swim/render/render3d.py` を修正する。

- 例:

- 過剰な設計変更は不要。

#### 期待挙動
- `motor.enable_switching=false` → 常に `RUN`
- `motor.enable_switching=true` かつ tumble相のとき → `TUMBLE`
- それ以外 → `RUN`
