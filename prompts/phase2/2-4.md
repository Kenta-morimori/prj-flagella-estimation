# Phase2 / 2-4 Codex Prompt: T ≡ η b^3 固定 + 論文表記つきログ強化（time-scale 可視化）

## 目的
Phase2-bead-rpy-core において、時間スケール合わせのために **モータートルク T を常に T = η b^3 と定義**する。
これにより論文の無次元化で用いる **τ = η b^3 / T が常に 1 [s]** となり、実時間とシミュレーション時間の感覚を揃える。
加えて、時間スケール関連ログを **論文中の変数表記を併記**してわかりやすく出力する。

## 適用範囲
- ブランチ：feature/phase2-bead-rpy-core
- 既存仕様（prompts/phase2/2-1.md, 2-3.md）は維持しつつ、**本プロンプトの指示を優先**する。
  - 特に 2-3 の「無次元時間 t* で回す」「dt_star をログ出力」などの枠組みは維持する。

---

## 重要：時間スケールの定義（本プロンプトで上書き）
### 1) トルクの定義を固定する
- これまで `motor.torque_Nm` を入力としていたが、**入力値は無視**し、起動時に必ず以下で上書きする：
  - `b_m = scale.b_um * 1e-6`
  - `eta = fluid.viscosity_Pa_s`
  - **`T_Nm = eta * (b_m ** 3)`**
- 以後の計算で使用するトルクは常に `T_Nm`（上記）とする。
- yaml上の `motor.torque_Nm` は互換のため残して良いが、ログで「上書きされた」ことが分かるようにする。

### 2) τ（秒）の定義
- 論文の定義に従い τ は
  - `tau_s = eta * (b_m ** 3) / abs(T_Nm)`
- ただし上記 1) より **常に tau_s = 1.0 [s] になる**（数値誤差を除く）。
- 無次元時間：
  - `t_star = t_s / tau_s`（論文表記：t*）
  - `dt_star = dt_s / tau_s`（論文表記：Δt* = Δt/τ）
  - `duration_star = duration_s / tau_s`（論文表記：t_end* = t_end/τ）

---

## 必須：ログ出力の強化（論文表記を併記）
起動時（INFO）に以下を **必ず** 1ブロックで出力する。既存の
`dt_star=..., dt_s=... s, duration_star=...`
に加え、論文変数表記を併記して読みやすくする。

### 出力項目（必須）
- `b_um`（b）
- `b_m`
- `viscosity_Pa_s`（η）
- `T_Nm`（T；ただし本実装では T=η b^3 と定義）
- `tau_s`（τ）
- `dt_s`（Δt [s]）
- `dt_star`（Δt* = Δt/τ）
- `duration_s`（t_end [s]）
- `duration_star`（t_end* = t_end/τ）
- `total_steps`

### ログ文字列フォーマット（例）
- 既存ログの置換/拡張として、次のように出す（表記はこの順序を推奨）：

`[time-scale] b=1.000000e+00 um, η=1.000000e-03 Pa·s, T=η b^3=1.000000e-21 N·m, τ=η b^3/T=1.000000e+00 s`

`[time-step ] Δt=dt_s=1.000000e-03 s, Δt*=dt_star=1.000000e-03 (=Δt/τ)`

`[duration  ] t_end=duration_s=4.000000e+02 s, t_end*=duration_star=4.000000e+02 (=t_end/τ), steps=4.000000e+05`

※ 倍精度の丸めで 1.000000e+00 から僅かにズレる場合はそのまま出してよい。

## 2-3 との整合（dt_star=1e-3 厳密ルールの扱い）
- 2-3 の「dt_star は必ず 1e-3」という整合チェックは **維持**する。
- 本プロンプトにより `tau_s ≈ 1` なので、結果的に
  - `dt_s ≈ 1e-3 [s]`
となる。
- したがって `time.dt_s` はユーザ入力として残して良いが、起動時に
  - `dt_expected = 1e-3 * tau_s`
  - `abs(dt_s - dt_expected) <= tol`
を満たさない場合は ValueError（2-3踏襲）。
- エラーメッセージには論文表記も併記する：
  - 「dt_star(=Δt/τ) must be 1e-3」

---

## 実装タスク
1. `tau_s` 計算に使うトルクを `T_Nm = eta * b_m^3` に固定し、既存 `motor.torque_Nm` を上書き（または参照しない）にする。
2. ログ出力を上記フォーマットに変更（論文表記を併記、Tとτも必ず出す）。
3. 既存ログ `dt_star=..., dt_s=..., duration_star=...` は削除せず、**同等情報を含むより読みやすいログへ置換**する（重複出力は避ける）。
4. 2-3 の dt_star 整合チェックを維持しつつ、エラー文言を論文表記付きに改善。
